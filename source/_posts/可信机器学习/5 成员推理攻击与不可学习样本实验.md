> 这篇笔记整理一次隐私攻击与不可学习样本实验。主线是先用模型输出行为做 Membership Inference Attack，再用 min-min 扰动构造让模型难以从干净语义中学习的训练集。

## 实验想回答什么

这组实验覆盖可信机器学习里两个相邻但目标不同的问题：

- 成员推理攻击（Membership Inference Attack, MIA）：给定一个样本和目标模型的输出，判断这个样本是否出现在目标模型训练集中。
- 不可学习样本（Unlearnable Examples）：在训练样本上加入有界扰动，让后续模型优先学习扰动捷径，从而在干净测试集上泛化失败。

前者是隐私攻击：模型本身可能泄露训练集成员关系。后者更像数据保护或数据投毒视角：通过改变训练数据，使未经授权的模型即使拿到数据也难以学到可泛化语义。

完整报告可以下载：[`可信机器学习成员推理与不可学习样本实验.pdf`](/files/可信机器学习成员推理与不可学习样本实验.pdf)

## MIA 的基本设定

成员推理攻击把目标模型当成一个会暴露行为差异的黑盒或灰盒。对同一个分类器来说，训练样本通常会表现为：

- 真实类别置信度更高；
- 预测分布熵更低；
- 交叉熵损失更小；
- top-1 与 top-2 类别概率 margin 更大；
- 更容易被分类正确。

因此攻击者不一定需要原始模型参数。只要能得到目标模型对样本的 logits 或 softmax 输出，就可以把这些统计量变成攻击特征，再训练一个二分类器预测 `in` 或 `out`。

## Target Model 与 Shadow Model

实验中 MNIST 被拆成 target model 和 shadow model 两条数据流。target model 用一部分训练图像作为成员样本，用测试图像作为非成员样本。shadow model 则用另一部分数据模拟目标模型的训练与验证过程。

影子模型的作用是生成攻击模型训练集：

```text
shadow model 在训练样本上的输出  -> label = in
shadow model 在验证样本上的输出  -> label = out
```

攻击模型学到的是“见过样本”和“没见过样本”的输出行为差异。训练好之后，再把它迁移到 target model 的输出上，判断 target 样本是否属于训练集。

这个流程的关键是保持 target 和 shadow 的数据切分一致，避免成员标签和图像切片错位。否则攻击准确率可能来自实现错误，而不是来自真实的隐私泄漏信号。

## 攻击特征与分类器

攻击模型不直接看 MNIST 图像，而是看目标分类器的输出行为。一个样本会被编码成如下特征：

```text
softmax probability vector
sorted softmax probability vector
true-class confidence
entropy
cross-entropy loss
top-1/top-2 confidence margin
whether prediction is correct
```

这些特征共同刻画模型是否“异常熟悉”某个样本。实验中可以用 LightGBM 这类梯度提升模型做攻击分类器；如果环境没有 LightGBM，也可以退回到 scikit-learn 的 histogram gradient boosting，保持 notebook 可执行。

评价时要用攻击样本总数作分母，而不是只用成员样本数量。MIA 本质是平衡二分类问题，指标分母写错会直接扭曲攻击准确率。

## 不可学习样本的 min-min 思路

不可学习样本希望在训练图片上加入一个小扰动 `delta`：

```text
||delta||_infty <= epsilon
epsilon = 16 / 255
```

扰动对人眼来说不应改变图像语义，但会诱导模型学习 shortcut。min-min 更新可以理解成两层循环：

1. 固定当前扰动，用加噪训练集训练 base model 若干步。
2. 固定 base model，对输入扰动做 PGD 更新，使训练损失进一步下降。

这里和对抗攻击的方向相反。对抗样本通常最大化损失，让模型预测失败；不可学习样本中的 min-min 扰动则最小化训练损失，让模型更容易拟合扰动捷径。结果是训练集看起来很好学，但从干净测试集上看，模型学到的东西不能泛化。

## Class-wise 噪声

Class-wise 设置为每个 CIFAR-10 类别维护一张共享扰动：

```text
noise shape = [10, 3, 32, 32]
```

每轮更新先用当前类别扰动训练 base model，再对训练集生成 image-wise `eta`，最后把同一类别样本的 `eta` 求平均，写回对应类别的共享噪声。

生成最终数据集时，每张图按真实标签取对应类别噪声，加到原始像素后再 clip 到合法范围。这里要特别注意索引：类别噪声应该用标签索引，而不是用样本下标索引。

## Sample-wise 噪声

Sample-wise 设置为每个训练样本维护独立扰动：

```text
noise shape = [50000, 3, 32, 32]
```

如果 dataloader 不 shuffle，就可以用 batch 起止下标定位每张图对应的扰动。每轮外循环先训练 base model，再对当前 batch 生成新的 `eta` 并写回 `noise[batch_start:batch_end]`。

构造不可学习数据集时，扰动需要保留正负号。正确做法是先把扰动转换到像素尺度，加到原始图像上，再对加噪后的图像 clip。若在相加前把扰动本身 clamp 到 `[0, 255]`，负扰动会丢失，min-min 效果会被破坏。

## 如何读实验结果

MIA 的核心指标是攻击准确率。若攻击模型能稳定高于随机猜测，说明目标模型输出中存在可利用的成员关系信号。攻击越强，通常意味着目标模型泛化间隙越明显，或者接口暴露了过多置信度信息。

不可学习样本的核心指标是用生成数据集重新训练模型后，在干净 CIFAR-10 test set 上的准确率。若 clean accuracy 明显下降，说明训练过程确实学到了扰动捷径，而不是原图语义。

这两个实验共同提醒我们：模型可信性不能只看普通测试准确率。一个模型可能准确率不低，却泄露训练集成员关系；一份数据也可能视觉上正常，却被扰动成对训练过程有害的样本。

## 复习抓手

可以把这组实验压缩成三句话：

- MIA 利用训练样本和未见样本之间的置信度、损失、熵等行为差异。
- Shadow model 用来合成 `in/out` 攻击训练数据，帮助攻击模型迁移到 target model。
- Unlearnable Examples 通过 min-min 扰动制造易学但不可泛化的 shortcut，class-wise 共享类别噪声，sample-wise 维护逐样本噪声。

如果和前一篇隐私笔记联系起来看，MIA 是隐私攻击的具体实现；如果和对抗样本联系起来看，不可学习样本同样依赖小扰动，但优化目标从“测试时让模型错”变成了“训练时让模型学偏”。
