# 数据驱动、学习式动画与蒙皮

> 这篇笔记把角色动画课程中偏“数据与表示”的三条线放在一起：先用动作捕捉和 motion matching 解释数据驱动动画如何从已有片段合成新动作，再看学习式方法如何把动作流形压缩成可生成、可控制的模型，最后回到蒙皮与表情动画，说明骨骼姿态如何真正变成屏幕上的网格形变。

## 1. 从关键帧到数据驱动动画

关键帧动画的基本流程是：动画师指定少数关键姿态，系统用样条、四元数插值等方法补出中间帧。它的优点是可控，但当角色需要在实时交互中响应复杂输入时，纯手工关键帧会遇到两个问题：

- 真实人类动作细节很难完全凭经验画出。
- 用户输入组合太多，不可能为每个状态手工准备一段动画。

数据驱动动画的核心假设是：真实动作位于高维姿态空间中的低维流形上。动作捕捉数据就是这个流形上的采样点。只要能检索、对齐、拼接和混合这些采样点，就可以生成看起来自然的新运动。

一帧角色姿态通常写成：

```text
pose = root position + root orientation + local joint rotations
```

其中 root 决定角色在世界中的位置和朝向，其他关节多用局部旋转表示。局部旋转便于跨角色复用，因为它不直接绑定世界坐标，而是描述“子骨骼相对父骨骼如何转动”。

## 2. 动作捕捉与动作重定向

动作捕捉（motion capture, MoCap）把真人运动数字化，常见方案包括：

| 方法 | 核心机制 | 优点 | 局限 |
|------|----------|------|------|
| 光学标记 | 多相机追踪反光 marker | 精度高，工业标准 | 场地复杂，怕遮挡 |
| 惯性 IMU | 传感器积分姿态与速度 | 便携，不依赖相机 | 长时间会漂移 |
| 无标记多目 | 多视角 2D 姿态到 3D | 演员负担小 | 精度通常低于标记系统 |
| 单目视频 | 单 RGB 视频估计 3D 姿态 | 获取成本低 | 深度歧义大 |

MoCap 数据不能直接套到任意角色上，因为不同角色的骨骼数量、骨长比例、默认姿态、命名和拓扑都可能不同。动作重定向（motion retargeting）要解决的就是：如何让源角色的动作语义迁移到目标角色。

一个典型流程是：

1. 建立源骨骼与目标骨骼之间的语义映射，例如左上臂、左前臂、手腕。
2. 对齐 bind pose / T-pose，消除默认姿态差异。
3. 将源角色的局部旋转映射到目标骨骼。
4. 用 IK 或约束修正脚滑、手部目标偏移、穿插等问题。

这里的难点不只是“复制旋转”，而是保留动作意图。一个高个角色和一个矮个角色执行同一段跨步动作时，脚掌落点、步幅和身体重心都需要重新协调。

## 3. 过渡、动作图与 Motion Matching

如果直接把两个动作片段首尾相接，角色会突然跳姿态或瞬移。过渡混合（transition blending）会先对齐 root，再对关节旋转做插值：

```text
new pose = blend(pose A, aligned pose B, weight)
```

其中位置可以线性插值，旋转通常用四元数插值，避免欧拉角带来的万向节锁和非最短路径问题。

Motion Graph 把动作片段组织成图：

- 节点是动作片段或动作帧。
- 边表示两个片段之间可以自然过渡。
- 在线控制时，在图上搜索一条满足目标的路径。

Motion Matching 更直接。它不预先构造复杂图，而是在每一帧把“当前角色状态 + 未来期望轨迹”编码成特征向量，在动作库中找最近邻帧，再从该帧继续播放。

常见特征包括：

- 当前关节位置、速度；
- 脚部接触状态；
- root 速度与朝向；
- 未来若干时间点的目标轨迹。

Motion Matching 的优点是工程上简单、效果稳定、响应实时；局限是强依赖动作库覆盖度。如果动作库里没有相近片段，检索出的运动就会不自然。

## 4. 学习式动画：从 motion prior 到生成模型

学习式动画把“动作流形”显式或隐式地学成模型。它关心的不只是从库里检索已有片段，而是要判断一个姿态是否自然、预测下一帧、或者生成全新动作。

最基础的线性模型是 PCA。给定大量姿态向量，PCA 找到方差最大的低维方向。一个姿态如果在这些主方向上的标准化偏移很小，就更像训练数据中的自然姿态；如果偏离很远，就可能是奇怪或不可能的姿势。

这可以写成一个 motion prior：

```text
naturalness(x) high  <=>  x close to learned motion manifold
```

神经网络方法用非线性模型替代 PCA：

| 方法 | 核心想法 | 适合解决的问题 |
|------|----------|----------------|
| 自回归网络 | 用上一帧和控制信号预测下一帧 | 实时角色控制 |
| PFNN | 用 gait phase 混合不同网络权重 | 周期性步态生成 |
| MANN | 用 gating network 自动选择专家 | 多风格、多动作切换 |
| VAE / VQ-VAE | 学低维连续或离散 latent code | 运动压缩与生成 |
| Diffusion | 从噪声逐步去噪成动作序列 | 文本到动作、离线生成 |

学习式方法的关键问题是误差累积：如果模型每帧只错一点，长时间 rollout 后角色可能漂移、脚滑或进入训练分布外姿态。因此实际系统通常会把学习模型与接触约束、物理仿真、动作库检索或后处理混合使用。

## 5. 从动作到网格：Linear Blend Skinning

骨骼动画本身只给出关节的位置和旋转；真正显示在屏幕上的是三角网格。蒙皮（skinning）负责把骨骼运动传递给网格顶点。

Linear Blend Skinning（LBS）的基本公式是：

```text
v'_i = sum_j w_ij * (R_j v_i + t_j)
```

其中：

- `v_i` 是 bind pose 下的顶点；
- `R_j, t_j` 是第 `j` 根骨骼当前相对 bind pose 的刚体变换；
- `w_ij` 是顶点 `i` 受骨骼 `j` 影响的权重；
- 权重通常非负，且对每个顶点求和为 1。

LBS 的优点是简单、高效、GPU 友好，所以游戏引擎和实时角色系统中大量使用。但它有一个经典缺陷：当关节大幅扭转时，多个旋转矩阵的线性组合不再是合法旋转，网格会塌陷成“糖纸”形状。这就是 candy-wrapper artifact。

Dual Quaternion Skinning（DQS）用对偶四元数表示刚体变换，在更接近 `SE(3)` 几何结构的空间中做混合，可以缓解大角度扭转导致的体积损失。

## 6. 表情动画与 Blendshape

面部动画通常不完全依赖骨骼，而是使用 blendshape。它把一组表情形变当作基：

```text
face = neutral + sum_k alpha_k * expression_k
```

其中 `alpha_k` 是每个表情基的权重，例如张嘴、眨眼、皱眉、微笑。FACS 和 ARKit 都可以看作对这些表情动作单元的标准化描述。

Blendshape 的优势是直观、可编辑、适合艺术家控制；局限是线性叠加可能产生不合理组合，因此复杂系统会加入约束、非线性校正或基于学习的表情模型。

## 7. 复习抓手

这组内容可以按“数据 -> 模型 -> 网格”来记：

- 数据驱动动画：MoCap 提供真实运动样本，retargeting 让样本适配不同角色。
- Motion Matching：把实时状态和未来目标变成特征，在动作库里找最合适的下一段。
- 学习式动画：把动作流形学成 motion prior、生成模型或控制策略。
- LBS：用骨骼变换的加权平均驱动顶点，快但会有糖纸伪影。
- DQS：用对偶四元数混合刚体变换，改善大旋转下的体积保持。
- Blendshape：用表情基线性组合生成面部形变，是面部动画的核心接口。

真正的角色系统往往不是单一技术，而是把这些层级串起来：MoCap 或生成模型给出姿态序列，控制器和约束保证它能响应任务，蒙皮与表情系统再把骨骼运动翻译成最终可见的角色表现。
