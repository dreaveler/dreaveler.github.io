> 这篇笔记接着“近似算法”的总览，专门展开两个经典例子：度量旅行商问题和 0/1 背包。前者展示三角不等式怎样把 NP 难问题变成可以常数近似的问题；后者展示一个 NP 难优化问题怎样从简单贪心一路做到 FPTAS。

## 为什么这两类问题值得放在一起看

近似算法的核心不是“跑得快但随便给个答案”，而是在多项式时间内输出可行解，并证明它离最优解的距离有最坏情况保证。度量 TSP 和背包分别代表两类常见分析方法：

- 对最小化问题，常用一个容易计算的结构作为最优解下界。例如 TSP 中最小生成树的权重不超过最优巡回路。
- 对最大化问题，常用一个放松问题或缩放后的精确算法控制损失。例如背包中分数背包给出上界，价值缩放控制取整误差。

这两类问题也说明“NP 难”并不自动等于“无法近似”。同样是 NP 难，度量 TSP 有常数近似，一般 TSP 没有常数近似；背包甚至有完全多项式时间近似方案。

## 度量 TSP 的前提：三角不等式

旅行商问题要求在给定城市距离后，找一条经过每个城市恰好一次并回到起点的最短回路。若距离函数满足

```text
d(u, w) <= d(u, v) + d(v, w)
```

就称为度量 TSP。三角不等式的意义是：从 `u` 直接走到 `w` 不会比绕经 `v` 更远。因此在一条经过重复顶点的行走路线中，如果某个城市已经访问过，可以“抄近路”跳过它，而且总长度不会增加。

这个条件是度量 TSP 可近似的关键。没有它，抄近路可能变长，下面所有基于欧拉回路的算法都会失去保证。

## 双生成树算法：MST 给出 2-近似

双生成树算法的步骤很直接：

1. 求一棵最小生成树 `T`。
2. 把 `T` 中每条边走两遍，得到一张所有顶点度数为偶数的多重图。
3. 在这张图上走欧拉回路。
4. 沿欧拉回路访问城市，遇到已访问城市就用三角不等式抄近路，得到一条 Hamilton 回路。

证明 2-近似只需要两条不等式。首先，最优 TSP 回路删掉任意一条边就是一棵生成树，所以

```text
w(T) <= OPT
```

其次，复制树边后得到的欧拉回路长度是 `2w(T)`。抄近路不会增长度，因此算法输出 `A` 满足

```text
A <= 2w(T) <= 2OPT
```

所以它是 2-近似算法。这个证明的结构很典型：先把最优回路和一个更简单的结构联系起来，再用这个结构构造可行解。

## Christofides 算法：只补奇度顶点

双生成树算法把整棵树加倍，显得浪费。Christofides 算法的改进点是：不把所有边复制一遍，而是只处理最小生成树里的奇度顶点。

算法步骤：

1. 求最小生成树 `T`。
2. 找出 `T` 中所有奇度顶点。根据握手定理，奇度顶点个数一定是偶数。
3. 在这些奇度顶点之间求一个最小权完美匹配 `M`。
4. 把 `M` 加到 `T` 上。此时所有顶点度数都变成偶数，于是可以走欧拉回路。
5. 再用三角不等式抄近路，得到 Hamilton 回路。

关键是匹配权重的上界。把最优 TSP 回路限制到这些奇度顶点上，并沿回路隔一条边取一条边，可以得到两个完美匹配。两个匹配的总权重不超过 `OPT`，所以较小的那个不超过 `OPT/2`。最小权完美匹配 `M` 当然不比它更重：

```text
w(M) <= OPT / 2
```

再加上 `w(T) <= OPT`，得到

```text
A <= w(T) + w(M) <= 3OPT / 2
```

这就是 Christofides 的 `3/2`-近似。它和双生成树算法的差别只在“怎样把图补成欧拉图”：前者复制整棵树，额外付出至多 `OPT`；后者只给奇度点做匹配，额外付出至多 `OPT/2`。

## 一般 TSP 为什么不能常数近似

如果取消三角不等式，一般 TSP 不存在常数近似算法，除非 `P = NP`。证明思路是把 Hamilton 回路问题归约到 TSP，并制造一个巨大的最优值间隙。

给定图 `G`，构造一个完全图上的 TSP 实例：

```text
如果 (u, v) 是 G 中的边，令 d(u, v) = 1
如果 (u, v) 不是 G 中的边，令 d(u, v) = Kn
```

其中 `K` 是某个常数近似比的上界，`n` 是顶点数。若 `G` 有 Hamilton 回路，则 TSP 最优值是 `n`。若没有 Hamilton 回路，任何巡回都必须至少使用一条非边，总代价会超过 `Kn`。

于是一个 `K`-近似算法就能区分“最优值等于 `n`”和“最优值大于 `Kn`”，从而判定 Hamilton 回路。这会推出 `P = NP`。因此一般 TSP 和度量 TSP 的可近似性差异，正是三角不等式带来的。

## 背包的简单 2-近似

0/1 背包给定每件物品的重量 `w_i`、价值 `v_i` 和容量 `B`，目标是在不超重的情况下最大化总价值。按单位重量价值 `v_i / w_i` 从高到低贪心是自然想法，但单独使用会被“大价值物品”击败。

一个简单修正是：

1. 按单位重量价值降序尝试装入物品，能装就装，得到贪心解 `G`。
2. 找出单个价值最大的可行物品。
3. 在两者中取价值更大的解。

这个算法至少能达到最优值的一半。直觉是，按性价比排序时，贪心解加上第一件装不下的物品，已经超过分数背包意义下的最优上界。算法又保证输出至少不小于“贪心已装价值”和“最大单件价值”中的较大者，所以最优值不会超过输出的两倍。

这个结论不精细，但很重要：它说明即使精确背包是 NP 难的，简单算法也能给出可证明的常数保证。

## PTAS：枚举少数大物品

背包的 PTAS 思想是：最优解中价值最大的少数物品影响最大，可以先枚举这些物品，再用贪心补全。

给定误差参数 `epsilon`，令

```text
m = ceil(1 / epsilon)
```

算法枚举最多 `m` 件物品作为起始集合，保留不超重的枚举方案，然后对剩余容量用单位价值贪心补全，最后取价值最大的结果。

若最优解本身不超过 `m` 件，枚举会直接猜中最优解。若最优解超过 `m` 件，算法至少会枚举到最优解中价值最大的 `m` 件。此时任何还没被选中的最优物品价值都不超过当前已选大物品总价值的 `1/m`，因此遗漏造成的相对损失可以控制在 `1 + epsilon` 内。

PTAS 的缺点是时间复杂度通常像

```text
n^(1/epsilon)
```

这样对 `1/epsilon` 呈指数依赖。当 `epsilon` 很小时，算法虽然理论上是多项式时间，但实际并不友好。

## FPTAS：价值缩放和伪多项式 DP

背包更强的结论是 FPTAS，即时间同时对输入规模和 `1/epsilon` 都是多项式。它依赖一个价值维动态规划。

先看精确 DP。令 `G_k(d)` 表示只考虑前 `k` 件物品，为了达到至少价值 `d` 所需的最小重量。递推是：

```text
G_k(d) = min(G_{k-1}(d), G_{k-1}(d - v_k) + w_k)
```

最后找出满足 `G_n(d) <= B` 的最大 `d`。这个算法对价值总和是多项式，但对输入的二进制长度不一定是多项式，所以叫伪多项式时间算法。

FPTAS 的做法是先把价值缩小并取整：

```text
K = epsilon * v_max / n
v_i' = floor(v_i / K)
```

然后在缩放后的 `v_i'` 上做价值维 DP。取整会让每件物品最多损失 `K` 的价值，整个解最多损失 `nK = epsilon * v_max`。又因为最大单件可行物品的价值不超过 `OPT`，总损失不超过 `epsilon * OPT`。

因此缩放后的精确解转回原价值后，至少达到

```text
(1 - epsilon) OPT
```

同时，缩放把最大价值压到和 `n / epsilon` 同阶，使动态规划时间变成关于 `n` 和 `1/epsilon` 的多项式。

## 复习抓手

把本讲放回近似算法主线，可以记住四个对比：

- 度量 TSP 能常数近似，一般 TSP 不能常数近似；差别是三角不等式。
- 双生成树算法用 `2w(T)`，Christofides 用 `w(T) + w(M)`，后者把额外成本从 `OPT` 降到 `OPT/2`。
- 背包从“贪心解和最大单件取较大者”得到 2-近似，再通过枚举大物品得到 PTAS。
- FPTAS 比 PTAS 更强，因为运行时间必须对 `1/epsilon` 也是多项式。

这两个例子共同说明：近似算法的难点不只是设计一个启发式，而是找到能和 `OPT` 建立关系的结构。TSP 靠 MST、匹配和三角不等式；背包靠分数上界、价值排序和缩放 DP。
