> 这篇笔记整理算法设计与分析课程中的近似算法部分。它接在 NP 完全理论之后，讨论当最优解很可能无法在多项式时间内求出时，怎样退一步设计高效算法，并用近似比证明“这个次优解最多差多少”。

## 为什么 NP 难之后还要学算法

NP 完全理论告诉我们，很多组合优化问题的精确最优解不太可能有多项式时间算法。例如最小顶点覆盖、旅行商问题、背包和调度问题的某些版本，都和 NP 完全判定问题紧密相关。

但这并不意味着这些问题在实践中只能放弃。近似算法的目标是把问题改成一个更现实的承诺：

```text
不保证求到最优解，但保证在多项式时间内求到一个可行解，
并且这个解相对最优解有可证明的最坏情况界。
```

近似算法关注的是最坏情况保证，而不是某些测试数据上的平均表现。一个算法若声称有 2-近似，就意味着任何输入实例上都不会比最优差超过 2 倍。

## 近似比怎样定义

设 `OPT(I)` 是实例 `I` 的最优目标值，`A(I)` 是算法 `A` 输出的目标值。为了统一最大化和最小化问题，近似比总写成大于等于 1 的形式：

```text
最小化问题：r_A(I) = A(I) / OPT(I)
最大化问题：r_A(I) = OPT(I) / A(I)
```

如果对所有实例都有 `r_A(I) <= r`，就说算法 `A` 是 `r`-近似算法。`r` 越接近 1，保证越强；`r = 1` 就是精确最优算法。

证明近似比通常有两步：

1. 找到最优值的下界或上界。由于 `OPT` 很难直接算出，证明中常用“平均负载”“最大单个作业”“最小生成树权重”“匹配大小”等容易分析的量作为桥梁。
2. 构造紧实例。若能构造一族输入使算法真的接近这个界，就说明分析出的近似比不是证明太松，而是算法本身在最坏情况下确实只能这样。

这个套路很重要：近似算法不是“凭感觉效果还行”，而是把可行解、最优值和一个可证明的不等式链绑在一起。

## 最小顶点覆盖：用匹配给出 2-近似

最小顶点覆盖问题要求在无向图中选择尽量少的顶点，使每条边至少有一个端点被选中。判定版是 NP 完全问题，所以我们不期待一般图上有高效精确算法。

一个简单的 2-近似算法是：

```text
V' = empty
while graph still has edges:
    choose any remaining edge (u, v)
    add both u and v to V'
    delete u, v and all incident edges
return V'
```

算法每次取一条边并加入两个端点。关键观察是，被选中的这些边两两没有公共端点，因为一旦取了 `(u, v)`，所有与 `u` 或 `v` 相关的边都会被删除。因此这些边构成一个匹配。

假设算法一共取了 `k` 条这样的边。算法输出大小是 `2k`。而任何顶点覆盖都必须覆盖这 `k` 条两两不相交的边，每条边至少需要一个不同顶点，所以最优解大小至少是 `k`。于是：

```text
algorithm / OPT <= 2k / k = 2
```

紧实例也很直接：如果图本身就是 `k` 条互不相交的边，算法会选 `2k` 个端点，而最优解只需要每条边选一个端点，共 `k` 个。这个例子说明 2 这个界对该算法是紧的。

## 多机调度：平均负载和最大作业是基本下界

多机调度问题给定若干作业和 `m` 台相同机器，每个作业有处理时间，目标是把作业分配到机器上，使最大机器负载，也就是完工时间，尽量小。

最自然的贪心算法是 list scheduling：按输入顺序处理作业，每次把当前作业放到负载最小的机器上。这个算法的近似比是：

```text
2 - 1/m
```

证明依赖两个任何调度都绕不开的下界：

- 最优完工时间至少是总处理时间除以机器数，也就是平均负载。
- 最优完工时间至少是最大单个作业的处理时间。

设贪心结果里负载最大的机器是 `M`，最后放到 `M` 上的作业是 `b`。在 `b` 被放上去之前，`M` 是当时负载最小的机器，所以它之前的负载不超过当时平均负载。再加上 `b` 的处理时间，并分别用“平均负载”和“最大作业”两个下界去估，就得到 `2 - 1/m`。

如果先把作业按处理时间从大到小排序，再做同样的贪心，就得到 LPT 或递降贪心。它的近似比改进为：

```text
3/2 - 1/(2m)
```

改进来自“先放大作业”。当最大负载机器上至少有两个作业时，它最后接收的那个作业不可能太大；否则前 `m+1` 个大作业会迫使某台机器在任何方案中承受至少两个这样大的作业，从而推出这个作业的处理时间不超过 `OPT/2`。这把普通贪心证明中的“最后作业 <= OPT”改成了“最后作业 <= OPT/2”，近似比也随之降低。

## 度量 TSP：MST 是下界，三角不等式允许抄近路

旅行商问题要求找一条访问所有城市并回到起点的最短回路。一般 TSP 不存在常数近似比算法，除非 `P = NP`；否则可以用近似算法区分图中是否有 Hamilton 回路。

课程中讨论的常数近似算法针对的是度量 TSP：城市距离满足三角不等式。这个条件非常关键，因为它保证“抄近路”不会让路径变长。

双生成树算法的思路是：

1. 求最小生成树 `T`。
2. 把每条树边复制一份，得到所有顶点度数为偶数的多重图。
3. 在这个图上走欧拉回路。
4. 遇到已访问城市时用三角不等式抄近路，得到 Hamilton 回路。

为什么是 2-近似？最优 TSP 回路删掉任意一条边就是一棵生成树，所以最小生成树权重 `w(T) <= OPT`。复制树边后欧拉回路权重是 `2w(T) <= 2OPT`。抄近路不会增重，因此最终回路也不超过 `2OPT`。

Christofides 算法把这个思路再省一半。它不复制整棵树，而是只给最小生成树中的奇度顶点补一个最小权完美匹配，使所有顶点度数变偶。匹配权重可以用最优回路限制在奇度顶点上的两组交替匹配来证明不超过 `OPT/2`，所以总权重不超过：

```text
w(T) + w(M) <= OPT + OPT/2 = 3OPT/2
```

这就是度量 TSP 的 `3/2`-近似。

## 背包：从简单贪心到 FPTAS

0/1 背包给定物品价值、重量和容量，要求在不超重的情况下最大化总价值。只按性价比贪心会失败得很严重：一个小而性价比略高的物品可能阻止算法选择一个价值巨大的大物品。

一个基础补救是把“性价比贪心解”和“单个价值最大的可选物品”取较大者。它能保证至少达到最优值的一半。直觉是：按性价比排序时，第一个装不下的物品 `k` 和已经装入的物品一起，给出了分数背包意义下的上界；因此“已装价值”和“物品 `k` 的价值”中至少一个达到这个上界的一半。

背包更重要的结论是它有 FPTAS，也就是完全多项式时间近似方案。核心方法是对物品价值缩放取整，然后在缩放后的价值上做动态规划。

令最大物品价值为 `v_max`，取：

```text
K = epsilon * v_max / n
v_i' = floor(v_i / K)
```

对缩放后的 `v_i'` 做按价值维度的 DP，时间会变成关于 `n` 和 `1/epsilon` 的多项式。每个物品因取整最多损失 `K`，总损失最多 `nK = epsilon * v_max <= epsilon * OPT`，所以最后得到 `(1 - epsilon)` 近似解。

这说明背包虽然精确求解是困难的，但它在可近似性上很友好：不仅有常数近似，还有 FPTAS。

## 复习抓手

近似算法这一讲可以按下面几条主线复习：

- 近似比是最坏情况乘性保证，不是实验平均值。
- 证明近似比时，先找 `OPT` 的可证明下界或上界，再把算法输出和它连接起来。
- 最小顶点覆盖的 2-近似用“匹配大小”做最优解下界。
- 多机调度的贪心分析用“平均负载”和“最大单作业”做最优值下界。
- 度量 TSP 的常数近似依赖两个条件：MST 是最优回路的下界，三角不等式保证抄近路不增重。
- 背包的 FPTAS 展示了“NP 难”不等于“不可高质量近似”。

从课程主线看，近似算法是 NP 完全理论之后的正面回答：既然许多最优问题精确求解困难，就转而问哪些问题还能高效得到可证明的好解，以及这个“好”可以严格量化到什么程度。
