> 这篇笔记接在 NP 完全理论的基础篇之后，重点整理 3-SAT 到顶点覆盖的构件归约、常见 NP 完全问题之间的关系，以及怎样把判定版复杂性结论用到搜索版和优化版问题上。

## 三种常用归约方法

证明一个问题 NP-hard 时，最重要的是选对归约方式。课程里可以把常见证明分成三类：

- 限制法：证明一个已知困难问题是目标问题的特殊情形。
- 局部替换法：按固定规则把原实例的局部结构改写成目标实例。
- 构件设计法：为变量、子句、路径或容量等对象设计 gadget，让目标问题的可行解和原问题的可满足性一一对应。

限制法最适合“目标问题更一般”的场景，例如 0-1 背包可以覆盖子集和的特殊情况。局部替换法更像语法改写，例如把 SAT 的长子句转成 3-SAT。构件设计法最常用也最容易考，因为它要求把一个逻辑约束翻译成图或组合结构里的选择约束。

## 顶点覆盖、独立集与团

给定无向图 `G = (V, E)`：

- 顶点覆盖要求每条边至少有一个端点被选中。
- 独立集要求被选中的点两两之间没有边。
- 团要求被选中的点两两之间都有边。

三者之间有非常直接的互补关系：

```text
S 是 G 的顶点覆盖
<=> V - S 是 G 的独立集
<=> V - S 是补图 G^c 的团
```

因此只要证明其中一个问题 NP 完全，另外两个通常可以通过补集或补图关系得到 NP 完全性。这个关系也提醒我们：看似不同的“覆盖边”“互不冲突”“两两相连”，本质上经常只是同一组选择约束的不同表述。

## 3-SAT 到顶点覆盖的构件归约

顶点覆盖是构件设计法的经典例子。给定一个 3-CNF 公式，有 `n` 个变量和 `m` 个子句，构造图 `G` 并设阈值：

```text
K = n + 2m
```

构造分三部分。

第一，给每个变量 `x_i` 建两个点 `x_i` 和 `not x_i`，并在二者之间连一条边。顶点覆盖必须从这条边里至少选一个端点。由于总预算刚好是 `n + 2m`，最后会被迫每个变量构件恰好选一个端点，这就对应一次真假赋值。

第二，给每个三文字子句建一个三角形。一个三角形的三条边至少需要选两个点才能覆盖，所以每个子句构件会消耗两个顶点预算，并留下一个未选点。

第三，把每个子句三角形中的文字点连接到变量构件中对应的文字点。这样，三角形里唯一没被选的文字点，其联络边必须由变量侧的同名文字点覆盖。

等价性证明的核心是预算下界。变量边一共需要至少 `n` 个点，`m` 个三角形一共需要至少 `2m` 个点，所以任何顶点覆盖都至少有 `n + 2m` 个点。阈值恰好设为这个数，就强制每个变量边选一个端点、每个三角形选两个端点。

如果公式可满足，就在变量侧选择为真的文字；每个子句至少有一个真文字，对应的联络边已经被变量侧覆盖，于是三角形里可以只选另外两个点。反过来，如果图有大小 `K` 的顶点覆盖，每个变量边上的唯一选择定义出一个赋值；每个子句三角形留下的那个未选文字点，其联络边必须由变量侧覆盖，所以这个文字为真，所有子句都满足。

## 常见 NP 完全问题谱系

掌握 NP 完全理论时，不必把每个困难问题孤立记忆，更有效的是记住常用归约源和问题谱系：

```text
SAT -> 3-SAT -> Vertex Cover
                  |
                  +-> Independent Set
                  +-> Clique

Exact Cover -> Subset Sum -> 0-1 Knapsack
                         \-> Bin Packing
                         \-> Two-machine Scheduling

Directed Hamiltonian Cycle -> Hamiltonian Cycle -> TSP
```

这些问题覆盖了几类常见结构：

- 布尔公式：SAT、3-SAT。
- 图选择：顶点覆盖、独立集、团。
- 路径巡回：哈密顿回路、TSP。
- 装包与求和：子集和、背包、装箱。
- 线性约束：整数线性规划。

遇到新问题时，可以先判断它更像哪一类结构。若问题在表达“选一些对象且覆盖所有约束”，常从 3-SAT、顶点覆盖或恰好覆盖入手；若问题在表达“若干数能否配平”，常从子集和入手；若问题在表达“是否存在一条访问所有对象的合法顺序”，常从哈密顿回路入手。

## 从判定版到搜索版和优化版

NP 完全性通常先针对判定问题，但实际问题往往是搜索版或优化版。课程后半部分强调了 Turing 归约的思路：如果能高效解决优化版，就能高效回答判定版；反过来，在很多离散问题里，如果能多次调用判定版，也可以通过二分或逐步固定选择来恢复最优值和最优解。

以调度问题为例，优化版问最小完成时间，判定版问是否存在不超过 `D` 的可行调度。若有优化算法，当然可以把最优值和 `D` 比较得到判定答案。若有判定算法，也可以在可能的 `D` 范围上二分，找到最小可行期限。

这类关系说明：

- 判定版难，通常意味着优化版不会更容易。
- 优化版能精确高效求解，通常会推出判定版也能高效求解。
- 搜索版、优化版和判定版之间的关系要写清楚使用的是 many-one 归约还是 Turing 归约。

## 复习抓手

做 NP 完全证明时，可以按下面顺序检查：

1. 先把目标问题写成判定版，并说明它在 `NP` 中。
2. 选择一个结构相近的已知 NP 完全问题。
3. 明确使用限制法、局部替换法还是构件设计法。
4. 给出多项式时间构造。
5. 分别证明 Yes 实例会映射到 Yes 实例，No 实例不会被误映射成 Yes 实例。

最容易失分的是归约方向和预算设置。要证明目标问题难，应从已知 NP 完全问题归约到目标问题；构件归约里的阈值通常不是随便设的，而是用来强制每个 gadget 只能按预期方式取最小可行解。
