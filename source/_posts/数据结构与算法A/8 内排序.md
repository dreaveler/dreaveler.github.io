# 基本概念
序列：线性表 由记录组成
记录：结点 进行排序的基本单位
关键码：唯一确定记录的一个或多个域
排序码：作为排序运算依据的一个或多个域


排序就是将序列中的记录按照排序码顺序排列起来的
内排序就是所有操作都在内存中完成


对于序列中记录的排序码  排序码不减称为不减序  不增称为不增序
正序与逆序：待排序序列满足排序要求即为正序   待排序序列翻转过来即满足排序要求即为逆序


排序的稳定性：序列中有排序码相同的记录  排序后相对次序不变
	稳定性的证明 形式化证明   不稳定性的证明通过反例证明


排序算法的衡量标准：
- 时间复杂度：记录的比较次数和移动次数
- 空间代价
- 算法本身的繁复度
# 插入排序(shell 排序)
## 直接插入排序
由前向后 每个记录向前找到自身的位置
稳定
空间：O(1)
时间：
- 最好：  比较：n-1次  移动：2(n-1)次  O(n)
- 最坏：比较:$O(n^2)$  移动：$O(n^2)$    $O(n^2)$
- 平均：$O(n^2)$
**对于短排序比较有效**
## shell排序
选用间隔序列 每次对间隔为n的子序列直接插入排序 直到间隔为1
若增量初始为n/2  每次/2  那么时间复杂度约为O(n\^2)
若选取适当的增量序列  可以使时间复杂度接近O(n)
不稳定
每次/2的增量序列  我们会发现后面一次的增量序列已经在前面排过序了  这样的效率并不高
### Hibbard增量序列
${2^k -1, 2^{k-1} -1, \ldots, 7, 3, 1}$<br>选取这样的增量序列 可以使得时间复杂度为$O(n^{3/2})$  选取其它的增量序列可以进一步减少


而shell排序的最好代价  增量序列应为 等于$2^p3^q$的序列  选到小于等于n的为止并将其倒序  这就是增量序列  可以达到$O(nlog_2n)$
# 选择排序
## 直接选择排序
选择未排序序列中最小的值并将其挪至第一位
不稳定
空间代价：Θ(1)
时间代价
- 比较次数：$Θ(n^2)$
- 交换次数：n-1
- 总时间代价：$Θ(n^2)$
## 堆排序
基于最大堆实现 效率更高
不稳定
建堆：$Θ(n)$
删除堆顶: $Θ(log n )$
一次建堆，n 次删除堆顶
总时间代价为 $Θ(nlog n)$
空间代价为$Θ(1)$
# 交换排序
## 冒泡排序
由n-1开始遍历 比较后不满足条件则交换顺序  直到有一次冒泡没有发生交换 说明排序完成
稳定
空间代价：$Θ(1)$
时间代价分析
比较次数
最少：$Θ(n)$
最多：$Θ(n^2)$
交换次数最多为 $Θ(n^2)$，最少为 0，平均为 $Θ(n^2)$
时间代价结论
最大，平均时间代价均为$Θ(n^2)$
最小时间代价为$Θ(n)$：最佳情况下只运行第一轮循环
## 快速排序
选取轴值 使得序列分为两个序列L R  L的排序码小于等于轴值  R的排序码大于等于轴值  然后对L R 分别再进行快排
这时分治的思想
在轴值的选择上  尽可能使得L R长度相等  我们可以选择最左边的值  随机选择一个值  选择平均值
划分方法为双指针的分割函数：
选择轴值并存储轴值
将最后一个元素放到轴值的位置
初始化i j  指向头尾
i递增直到遇到比轴值大的元素 此元素放至j指针所指  j递减直到遇到比轴值小的位置 此元素放到i指针所指
重复上一步直到i==j  轴值放到i/j的位置 结束
不稳定
时间代价：解递推方程
最坏情况为O(n\^2)   最好情况为O(nlogn)   平均情况为O(nlogn)
空间代价：
最坏为O(n)  最好为O(logn)  平均为O(logn)
# 归并排序
划分为两个子序列 对两个子序列归并排序 有序子序列合并
稳定
先划分 再合并 合并时完成排序
合并时的排序使用双指针 完成排序
### 优化
可以对已基本排序序列直接插入排序
R.Sedgewick 优化：归并时从两端开始处理，向中间推进，简化边界判断


优化的归并排序
设置一个阈值 当序列长度小于阈值时 使用直接插入排序  大于时 使用优化的归并排序
将序列分为两个子序列  再进行合并
合并时 将l序列copy仅temparray   将r序列反向copy进temparray
再从temp array的两端用双指针完成排序


空间代价为O(n)
时间代价最好最坏平均均为O(nlogn)
# 分布排序和索引排序
不需要进行记录之间的两两比较 但需要提前知道序列的一些情况
## 桶式排序
事先知道序列中的记录都位于一个小区间内  $[0,m)$
将拥有相同值的记录都分配到一个桶 然后依次按照编号从桶中取出记录 组成一个有序序列


对于每个值中有几个数字进行count  然后累加得到这个值前一共有几个数字  然后从序列的尾部开始向前遍历(这样可以保证稳定性) 根据值来判断应该在第几位  完成排序


数组长度为 n, 所有记录区间  \[0, m) 上
时间代价：
统计计数：Θ(n+m) ， 输出有序序列时循环 n 次
总的时间代价为 Θ(m+n)   适用于 m 相对于 n 很小的情况
空间代价：
m 个计数器，长度为 n 的临时数组，Θ(m+n)
稳定
## 基数排序
将值部分拆分为多个部分进行桶式排序
例如对于整数来说 按位分解 先对个位桶式排序  对十位桶式排序 。。。以此类推  最后得到的结果就是排序好的结果  也可以高位先排 但方法有所不同


如果低位先排 则需要稳定排序  低位排好 合在一起 高位再排 再合 是一个分合分合的过程  比较简单计算机常用  LSD
高位先排 使用分治递归  分为不同桶  再在不同桶中 按低位分为小桶  直到最低位 最后再合并得到的就是排序好的序列  是一个 分分分分收的过程  MSD


实现方式 既可以使用顺序表来实现桶排序  也可以使用链表来实现
顺序表的实现中 就是不断的覆写count数组以及为了排序创造的临时数组  空间代价为O(n+r)
时间代价为 d次桶式排序
链表实现中 就是每个值作为链表的头 每次等于值的元素就链在链表的后面 这样避免了多余的空间消耗  只需要保存序列的头尾指针  空间代价为O(n+r)  时间代价中 不需要进行覆写 只需要修改next指针


时间代价为 Θ(d·n)，  实际上还是 Θ(nlog n)  没有重复关键码的情况，需要 n 个不同的编码来表示它们  也就是说，$d >= log_rn$，即在 Ω(log n) 中
## 索引排序
不修改原先数组  而是在索引数组中进行排序  一般的排序方法都是适用的
也就是说最后想要调用排序好的数组应该是Array\[IndexArray\[i\]\]
第二种方法是索引保存排序好的数组想要回到原数组的索引 即排序好是Array\[i\]  而如果想要获得原数组 是Array\[IndexARRAY\[i\]\]
# 排序算法的时间代价
一个长度为n的序列的逆序对平均有 n(n-1)/4组  因此任何只对相邻记录进行比较的方法 时间复杂度都是O(n\^2)
排序问题的时间代价在 O(n) (单趟扫描) 到 O(n log n) (平均, 最差情况) 之间<br>基于比较的排序算法的下限也为O(n·log n)<br>
所有排序算法都需要 O(n· log n)的运行时间，因此可以推导出排序问题的时间代价为 Θ (n· log n)
