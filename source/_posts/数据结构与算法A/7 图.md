# 定义与术语
G(V,E)   V是vertex的集合   E是edge的集合
有完全图 稀疏图(稀疏度  边条数小于完全图的5%) 密集图之分
无向图：边涉及顶点的偶对无序  实际上就是每对点之间是双连通的
有向图：边涉及顶点的偶对有序
标号图：边或顶点带有标签
带权图：边或顶点赋予权重
顶点的度：与顶点相关联的边的数目 有入度和出度之分
子图：子图的顶点与边都位于图中 且E'连接的v都在V'中 那么称为子图
路径：由顶点Vp到Vq的序列   简单路径与路径长度
回路：起点与终点都相同  简单回路就是序列中没有相同的点  有向无环图(DAG)
- 无向图中 由于起点与终点间有两条边  很容易被看作环 但它不是环  无向图中的环路径长度需要\>3
- 有向图中(V0,V1)与(V1,V0)可以构建为一个环


有根图：如果一个图中 从一个顶点可以到达所有的顶点 那么我们称它为图的根
连通图
- 对无向图：从V1到V2有一条路径 那么V2到V1也一定有一条路径 那么我们称它们是连通的
- 对有向图：从V1到V2有一条路径且V2到V1也有一条路径 那么称它们是强连通的
	- 如果所有的点对之间都是强连通的  那么称它是以一个强连通图
	- 而非强连通图中的极大强连通子图 称为强连通分量
		- 强连通分量可以只有一个点
网络：带权的连通图  这里的权是边上的

# 存储结构
## 相邻矩阵/邻接矩阵(adjacency matrix)
表示顶点之间的邻接关系 即有没有边
设G是一个有n个顶点的图  那么矩阵为(n,n)  定义为

$$
A[i, j] =
\begin{cases}
1, & \text{若 } (v_i, v_j) \in E \ \text{或} \ \langle v_i, v_j \rangle \in E \\
0, & \text{若 } (v_i, v_j) \notin E \ \text{且} \ \langle v_i, v_j \rangle \notin E
\end{cases}
$$

对于n个点的图 空间代价均为$O(n^2)$  与边无关
对于无向图 它是对称的  如果边有权  A(i,j)=权即可
可以计算稀疏因子$\delta$  $m\times n$ 的矩阵中 有t个非零向量   稀疏因子等于$\frac{t}{m\times n}$
若 $\delta < 0.05$   可以认为是稀疏矩阵


对于稀疏图 用邻接矩阵会存储大量0  浪费大量存储空间时间  我们采用邻接表 这样的链式存储结构
## 邻接表
对于每个顶点 记录它所连接的边
顶点结点记录为 data 与下一个指针
边结点记录为  可能记录的数据/权  目标节点的序号 下一个边的指针
![](/images/notes/数据结构与算法A/c82f2d05e857-01.png)
对于有向图来说 邻接表记录的是由当前结点出发的edge
与之对应的称为逆邻接表  即target结点为当前结点的edge
空间代价：n个顶点e条边的无向图 记录n+2e个结点  有向图 n+e个结点
e较小时 可以i节省大量存储空间  边表中表目顺序一般按顶点编号从小到大排列
## 十字链表
可以看为是邻接表与逆邻接表的结合
它由两种结点组成 一种为弧结点 一种为顶点结点
弧结点中包含：头 尾 下一条共尾弧 下一条共头弧  可能有的权(info域)
顶点结点包含：data域 第一条以该结点为终点的弧  第一条以该结点为头的弧
# 遍历
结点的遍历 每个结点访问且仅访问一次
必须考虑到：非连通图从一个结点可能无法到达所有结点  环的存在
解决方法 设置一个flag  当访问过后 flag变为已访问
## DFS
- 设置一个v0为原点 访问v0
- 重复访问v0的下一个元素 直至下一个元素均被访问完
- 返回 访问v0的其它元素 重复
- 当v0访问结束 以其它没有被访问的结点为原点 继续访问
```javascript
void DFS(Graph& G, int v)  { 	// 深度优先搜索的递归实现
	G.Mark[v] = VISITED;      		// 把标记位设置为 VISITED
	Visit(G,v);			       			// 访问顶点v
	for (Edge e = G.FirstEdge(v); G.IsEdge(e); e = G.NextEdge(e))
		if (G.Mark[G.ToVertex(e)] == UNVISITED)
			DFS(G, G.ToVertex(e));
		PostVisit(G,v);   // 对顶点v的后访问
}
```
## BFS
优先访问v0的邻接点
使用队列进行


对于DFS和BFS来说 每个顶点访问一次 每条边处理一次 无向图的每条边都需要从两个方向处理
采用邻接表表示时 无向图为O(n+2e)  有向图为O(n+e)
采用邻接矩阵时 为O(n\^2)
## 拓扑排序
有向无环图中，V里顶点的线性序列称作一个拓扑序列 该顶点序列满足若G中从顶点vi到vj有一条路径 则在序列中vi在vj之前
拓扑排序：将一个有向无环图中所有顶点在不违反先决条件关系下 排成线性序列的过程称为拓扑排序
算法：
- 从图中任意选择一个入度为0的顶点并输出
- 从图中删除该顶点以及所有的出边 将其入度-1
- 回到第一步反复执行
实现
- 可以使用队列实现
	- 入度为0的点入队
	- 若队列非空 则取出队首的元素 访问 进行操作
	- 若顶点入度减为0则入队
	- 判断图中是否有环：在队列为空后判断 如果G有未访问的结点 那么说明G有环
- 使用深度优先搜索
	- 先设置一个result数组  它用于存储结点idx
	- 对于未访问的结点 先对于所有的相邻结点进行递归 再把这个结点存入result数组
	- 再全部记录入result后  逆序返回
	- 逆序visit
拓扑排序的时间复杂度与图的DFS相同
# 最短路径
## 单源最短路径
带权图中 所有的权都≥0 对于一个顶点v0  它到所有其余顶点的权最小的最短路径
### Dijkstra算法
基本思想：把所有的点分为两组 第一组包括以确定最短路径的点 第二组是其余的点，按照最短路径长度递增的顺逐个把第二组的结点加到第一组中 直到从s出发可达的结点都包括进第一组中
不断维护dist数组即可   可以通过维护优先队列来优化算法
时间代价分析：每次改变D\[i\].length  在最差情况下 堆中数目为Θ(\|V\|)增加到Θ(\|E\|) 时间复杂度为Θ((\|V\|+\|E\|) log\|E\|)<br>它对于非连通图 不可达 但没有必要修改
不支持负权值
## 每对结点间最短路径
可以以每个结点为起点 调用n次Dijkstra算法
### Floyd算法
用相邻矩阵adj来表示带权有向图
算法
- 初始化adj0为相邻矩阵adj
- 在adj0上做n次迭代 递归产生一个矩阵序列 adj1  adj2.。。
- 经过k次迭代 得到的$adj^{(k)} [i,j]$的值为结点i到j路径上经过的结点序号不大于k的最短路径长度
- 本质是dp
第k次迭代时 若不经过k  则$adj^{(k)}[i,j] = adj^{(k-1)}(i,j)$
若经过k 则 $adj^{(k)}[i,j] = adj^{(k-1)}(i,k)+ adj^{(k-1)}(k,j)$
三重for loop 时间复杂度为O( $n^3$)
# 最小生成树
图G的生成树是一棵包含G所有结点的树   而最小生成树就是其中代价最小的一棵树 简称为MST
## Prim算法
也是贪心法
先从任意一个顶点开始 然后对于所有连接的但另一端顶点不位于MST中的边 选择代价最小的 将其加入MST 直到所有顶点都加入MST
最后得到的MST 由n个点以及n-1条边
可以用反证法证明它的性质是正确的
时间复杂度：O(n\^2)  因此适用于稠密图  对于稀疏图 类似的可以用最小堆来优化
## Kruskal算法
首先将G的n个顶点视作独立的n个连通分量  然后选择E中代价最小的edge  若其连通的是两个不同的连通分支 则将E加入MST  否则跳过至下一个
直到所有的顶点都位于一个连通分量中 就构建好了MST
可以理解为等价类的合并与压缩 这其中也就可以使用路径压缩
时间代价接近于O(n loge)
