问题——算法——程序
问题是一个函数 由输入到输出的函数
算法，指令的有限序列，一种方法，对特定问题求解的描述
### 数据结构
结构：实体+关系
数据结构：
- 按照逻辑关系组合起来的数据
- 按一定储存方法进行存储
- 在数据上定义了一个运算的集合


数据结构的逻辑组织：
- 线性结构
	- 线性表(表 栈 队列 串 etc)
- 非线性结构
	- 树
	- 图
例：线性结构的特点
- 有一个唯一的开始节点 无前驱  有一个唯一的后继
- 有一个唯一的终止节点
- 其它均成为内部节点
- 节点间的前驱/后继关系 有反对称性和传递性


数据的存储结构
每一个结点对应唯一的连续存储区域
四类：顺序 链接 索引 散列


抽象数据类型 简称ADT  定义了一组运算的数学模型 与物理存储结构无关 使软件系统建立在数据上  模块化的分装


先定义逻辑结构 再定义运算
### 算法
算法的特性
- 通用性
- 有效性
- 确定性
	- 下一步执行步骤必须明确
- 有穷性
	- 应在有限步内结束


基本算法分类
- 穷举法
	- 顺序找K值
- 回溯/搜索
	- 八皇后 树 图遍历
- 递归分治
	- 二分找K值  快排 归并排序
- 贪心
	- Huffman 编码树  最短路Dijkstra算法  最小生成树Prim算法
- 动态规划
	- 最短路 Floyd算法
# 算法的效率度量
数据规模n增大时 只与幂次最高的式有关
### 大O表示法
表示算法的效率上限 当上下限相同时可用O中加一杠来表示
当n足够大时 c g(n) ≥ f(n)
单位时间：
- 简单布尔/算数运算
- 简单I/O
- 函数返回
运算法则：
- 顺序结构 if  switch结构：加法规则 即时间复杂度选取大的一个
- for while do-while ：乘法规则 即时间复杂度为乘积


### Ω表示法
表示的是最大的下限


需要考虑到时间与空间开销
增大空间开销可能改善时间开销


下面是一个例子：
如果我们想获得一个长度为n  每个元素右移k个位置的新数组  且只能用一个单位的空间开销  可以怎么做：
- 首先我们会想到保存一个原始位置的元素  然后对于每个元素进行右移  但这样的时间复杂度是O(k n)
- 然后我们会想到如果可以有k个单位的空间开销 我们就可以直接把k个元素保存在原数组后面 然后直接提取这个temp数组
- 那么这里就提出了**海豚算法**
```c++
void Dolphin(int *Array, int n, int k) {
   int i,j,d;   int X;
   k = k % n;  if (k == 0) return;  	// 边界
   d = gcd(n,k);   				// d为n和k的最大公约数
   for (i=1; i<=d; i++) {
        X = Array[n-i];  			// 先保存最尾部的元素
        // 步长为k循环，视数组为环状。
        for (j=n-i-k; j!=n-i; j=(n+j-k)%n)
              Array[(j+k)%n] = Array[j];  // 移动
        Array[(n-i+k)%n] = X;   		// 补回x
  }
}
```
- 此外 我们还可以想到 如果我们将k前的数组部分置逆 再与后面的数组部分拼接起来  也成为了新的数组 我们就提出了置逆算法
