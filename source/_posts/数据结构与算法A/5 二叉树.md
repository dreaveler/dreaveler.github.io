# 基本概念
结点的有限集合  要么为空集  要么为有一个根节点与两个互不相交的左子树与右子树构成
左子节点与右子节点是可能有物理语义的 比如左边就是右边大之类的
## 五种基本形态
空 独根  空左  空右   左右都不空
3和4代表两棵不一样的树
## 基本概念
父节点 子节点 最左节点  兄弟节点(有同一个父节点的)   分支节点  叶节点
边 路径(由根到指定节点的长度)  祖先 后代
层数 (根节点是第0层) 深度 最大的层数  高度 深度+1
满二叉树：任何节点 要么是叶节点  要么是恰有两棵非空子树
完全二叉树：从上到下 从左往右 依次摆
完全二叉树可以用顺序表来表示
扩充二叉树：把一棵任意的树扩充为满二叉树  原来的叶节点 也扩充两个子节点
外部路径长度与内部路径长度：所有叶子节点/分支节点的路径长度的和
扩充二叉树满足一个关系： $E = I + 2 \times n$
n是内部节点个数  使用数学归纳法进行证明
## 主要性质
- 性质1. 在二叉树中，第i层上最多有 2i 个结点（i≥0）
- 性质2. 深度为 k 的二叉树至多有 2k+1-1个结点（k≥0）
	- 其中深度(depth)定义为二叉树中层数最大的叶结点的层数
- 性质3. 一棵二叉树，若其终端结点数为 n0，度为2的结点数为 n2，则 n0=n2+1
- 性质4. **满二叉树定理**：非空满二叉树树叶数目等于其分支结点数加1
- 性质5. 满二叉树定理推论：一个非空二叉树的空子树数目等于其结点数加1
- 性质6. 有n个结点（n\>0）的完全二叉树的高度为** **log2 (n+1) ** (向上取整)**（深度为log2 (n+1) ** **- 1）
所有的二叉树，我们可以分为N = n0 + n1 + n2   下标代表度数    度数为有几个子节点
边的数量E = n-1   而我们也可以这么计算   $E = 0 \times n_0 + 1 \times n_1 + 2 \times n_2$

## 深度优先遍历
```c++
访问根节点  \\前序
访问左子节点
访问根节点  \\中序
访问右子节点
访问根节点  \\后序
```
对于一个表达式二叉树来说 三种访问方式分别对应前缀、中缀、后缀表达式
前序+后序不能唯一确定树的结构  但中序+前序/后序可以
### 非递归DFS
用栈模拟
前序模拟：
```c++
栈S,x=root
while(x):
	访问x
	if(x->rightchild != NULL) S.push(x->rightchild)
	if(x->leftchild != NULL) x=x->leftchild)
	else x=S.pop()
```
思想：先访问当前节点 并把右子节点压入栈  当前节点变为左子节点  左子树遍历结束后从栈顶pop
中序模拟：
```c++
栈S,x=root
while(x||!S.empty()):
	if(x) S.push(x) x=x->leftchild
	else x=S.pop()
			访问x
			x=x->rightchild
```
思想：把当前节点先压入栈 并访问左子节点  直到左子树全部压入栈 栈顶pop并访问   然后访问右子节点
后序模拟：
由于需要在两次遍历后返回 无法分辨 因此需要加上一个特征表示位  Left与Right表示是左子树返回与右子树返回
```c++
栈S,x=root
while(x||!S.empty()):
	while(x):
		x.tag=left
		S.push(x)
		x=x->leftchild x=S.pop()
	if(x.tag=Left)
		x.tag=right
		S.push(x)
		x=x->rightchild
	else 访问x x=NULL
```


时间复杂度分析：
每个节点都被访问且访问1次  O(n)
空间复杂度：最好O(logN)  最坏O(N)  栈的深度取决于树的高度
## BFS遍历

完全二叉树右自身的下标公式：<br>从节点的编号就可以推知父/兄弟/子节点编号
- 当 2i+1\<n 时，结点 i 的左孩子是结点 2i+1，否则结点i没有左孩子
- 当 2i+2\<n 时，结点 i 的右孩子是结点 2i+2，否则结点i没有右孩子
- 当 0\<i\<n 时，结点 i 的父亲是结点 (i-1)/2 (下取整)
- 当 i 为偶数且 0\<i\<n 时，结点 i 的左兄弟是结点 i-1，否则结点 i 没有左兄弟
- 当 i 为奇数且 i+1\<n 时，结点i的右兄弟是结点 i+1，否则结点i没有右兄弟


# 二叉搜索树(BST)
空树或具有这个性质的二叉树：对任一节点 v为K 左子树的任意一个节点的值小于K  右子树任意一个节点的值大于K  左右子树也为BST
性质：中序遍历是正序的  由小到大排列
检索一个值 只需要检查两个子树之一  直到找到  若没找到 就是无
插入一个值 先检索这个值 找到则不插入  没找到则在该位置插入新叶  保持BST的性质和性能
删除：找到这个值的节点 如果左/右空，则将这个节点的唯一子节点设为这个节点，再删除 否则：
找到右子树的最小节点  然后将这个节点的值设为当前节点  并删除这个节点 由于它要么没有节点 要么只有右子节点 所以可以很方便的完成(大于该结点的最小结点)


二叉搜索树的总结：它可以组织内存索引 是很重要的一种树形索引
# 堆与优先队列
## 堆的定义
最小堆：
对于其中的节点从上到下 从左到右依次排序  是一个关键码序列
其具有如下性质： $K_i <= \min(K_{2i+1},K_{2i+2})$  类似的可以定义最大堆
用完全二叉树实现  其层次顺序可以用数组实现  具有局部有序性 节点只与子节点有关系 与兄弟节点无直接限制，实际上是一种树形结构
siftdown是把上面的小值与下面小的一个交换
```c++
Siftdown(int position){
	i = position;
	j = 2 * i + 1;
	T temp = Array[i];  //保存初始值
	while( j < currentsize){
	if(j < currentsize-1 && Array[j] > Array[j+1]){
		j++;   //选择较小的一个子节点
	}
	if(temp > Array[j]){
		Array[i]=Array[j];  //把较小的值交换上去
		i = j;
		j = 2 * j + 1;  //继续向下
	}else break;
}
Array[i] = temp;//把初始值赋到下面
}
```
```c++
void MinHeap<T>::SiftUp(int position)  {
    // 从position向上开始调整，使序列成为堆
    int temppos=position;
    // 不是父子结点直接swap
    T temp=heapArray[temppos];
    while((temppos>0) && (heapArray[parent(temppos)] > temp))   {
             heapArray[temppos]=heapArray[parent(temppos)];
                temppos=parent(temppos);
    }
    heapArray[temppos]=temp;// 找到最终位置
}
```
Siftup是从底向上捋的过程  把大值与上面小的交换
## 操作
### 建堆
对于前n/2个元素 从右到左 依次使用siftdown
### 插入
把节点插入最后一个位置  然后siftup
### 删除
把最后一个元素的值赋到要删除的这个节点处 然后与父节点比较 siftup/down
```c++
bool MinHeap<T>::Remove(int pos, T& node)  {
    if((pos<0)||(pos>=CurrentSize))
        return false;
    T temp=heapArray[pos];
    heapArray[pos]=heapArray[--CurrentSize];
    if (heapArray[parent(pos)]> heapArray[pos])
        SiftUp(pos);	//上升筛
    else SiftDown(pos); 	// 向下筛
    node=temp;
    return true;
}
```
### 时间效率
建堆为O(n)  其余操作平均与最坏均为O(logn)


堆可以用于实现优先队列
# Huffman树
## 编码
计算机的二进制编码 ASC2码  中文编码
首先有等长编码  假设所有字符的编码都等长  表示n个不同字符需要log2n位  字符的使用频率相等
数据压缩与不等长编码
对于出现频率较高的字符编码较短  根据出现频率来编码
数据压缩既可以减少硬盘存储  也可以加快读取速度
**前缀编码：任何一个字符的编码都不是另外一个的前缀 这样可以保证是一个双射**
## Huffman树
将代码与字符相联系 是不等长编码  代码长度取决于对应字符的相对使用频率或”权”
### 构建huffman树
对于n个字符 它们有各自的出现频率(权) $w_i$  需要给出前缀编码使得编码效率最高
给出具有n个外部节点的扩充二叉树  每个外部节点有一个权 也有一个路径 $l_i$
叶节点的带权外部路径总和为 $\sum w_i l_i$  权越大离根节点越近
步骤
- 按权排为一列  取出最小的两个标记为树叶  而这两个是一个分支节点的子节点  它的权是这两个子节点之和  将这个权放回原先的排列
- 重复步骤直至处理结束
从根节点中 左边分支标为0 右边标为1 字符的huffman编码就是从根节点到叶子节点的二进制编码串  将其放入表中 对字符串进行编码时 就可以通过查表完成
### 译码
过程与构建相反  0则进入左子节点 1进入右子节点  到达树叶节点后 对应的字符就是文本信息的字符
连续翻译则译出一位后回到根节点从二进制位串的下一位开始译码
## 性质
引理：含有两个节点以上的huffman树 字符使用频率最小的两个字符是兄弟结点，而且其深度不比树中其他任何叶结点浅
定理：对于给定的一组字符，函数HuffmanTree实现了“最小外部路径权重
证明使用数学归纳
平均每个编码的长度为：(c0f0 + c1f1 + … + cn-1fn-1) / fT


适合于： **字符频率不等 差别较大的情况**
应用：将百分制划为五分制  使得程序中比较次数最少  根据成绩分布选取二叉树顺序
