# 线性表
## 逻辑结构
- 长度
- 表头 head
- 表尾 tail
- 当前位置 cur_pos
### 根据存储分类
- 顺序表
	- 一般用数组实现
	- 按索引值从小到大放在相邻区域 紧凑结构 存储密度为1
- 链表
	- 单链表
	- 双链表
	- 循环链表
### 按操作分类
- 线性表
	- 不限制操作
- 栈
	- 在同一端操作
- 队列
	- 在两端操作
下面是一个线性表类的模板
```c++
template<class T>
class List{
	void clear();
	bool isEmptyy();
	bool append(const T value);   //传入成功后返回true
	bool insert(const int p,const T value);
	bool delete(const int p);
	bool getPos(int& p,const T value);
	bool getValue(const int p,T& value);
	bool setValue(const int p,const T value);
}
```
此外实际实现的时候应该更多的是传引用或者指针 毕竟直接复制一个值的开销还是太大了
# 顺序表
也称向量 固定长度的一维数组
$Loc(k_i) = Loc(k_0) + i \times c,\quad c = sizeof(Element)$
## 定义
```c++
class arrList : public List<T>{
private:
	T* aList;
	int maxSize;   //指的是定义时开的内存最大长度
	int curLen;    //当前的长度 动态变化  但不超过maxSize
	int position;
public:
	int length();
	bool append(const T value);   //不能超过maxSize
 ...
 }
```
有趣的是  对于动态数组 vector.etc  如果在append时超过了当前的maxSize  一般会重新开一个maxSize是两倍的数组  再把当前的这个搬过去  然后释放内存
### 插入和删除
这两种操作的代价主要体现在表中元素的移动
插入：移动n-i个
删除：移动n-i-1个
如果我们假设在顺序表每个位置上杀入和删除元素的概率相同 那么就可以计算插入和删除的平均技术 最后我们可以得出这两种操作的时间复杂度都是O(n)
### 优点
不需要附加空间
随机存取任一元素
### 缺点
很难估计所需空间大小
# 链表
- 存储利用指针
	- 动态的按照需要为表中新的元素分配存储空间
分类
- 单链
- 双链
- 循环链
一般链表的头节点不会存储数据  如果头节点不存储数据  那么head指针是None  那么还需要进行一个pre-judge  极其容易出错
### 单链表的运算
对于任一节点 都必须从第一个点开始 再找到这个节点
时间复杂度 O(n)
对于插入和删除操作是加上了一个O(1) 可以省略
### 双链表的插入
**注意顺序**
- 创造新节点q
- 绑定q的next节点
- 绑定q的prev节点p
- 重绑p的next节点为q
- 重绑q的next节点的prev节点为q
这里如果指针学的比较明白就看一眼就能明白
### 删除
删除p节点
重绑p的prev节点的next节点为p的next节点
重绑p的next节点的prev节点为p的prev节点
设定p的next节点与prev节点为None


如果后面p还有需要可以先不删除 实际合适时再删除
