> 这篇笔记整理大语言模型的后训练主线：为什么只做预训练的 base model 还不是一个可用助手，监督微调、奖励建模、RLHF 和 RLAIF 分别解决什么问题，以及 InstructGPT 式流水线为什么成为 ChatGPT 之后常见的对齐范式。

## 为什么预训练之后还要后训练

预训练语言模型的基本目标是下一词预测：给定前文，预测下一个 token。这个目标让模型从大规模语料中学到语言模式、事实知识、文体习惯和一定的推理能力。GPT-2、GPT-3 之后，模型还表现出少样本上下文学习能力：只要 prompt 中给几个例子，模型就可能照着模式完成新任务。

但这还不等于“会听指令”。一个只做预训练的 base model 本质上是在补全文本，而不是在理解用户意图后执行任务。用户说“给我一段摘要”，实际可能隐含很多控制条件：给五年级学生看的摘要、适合发社交媒体的一句话、面向研究者的一页综述，或者一段只保留结论的执行摘要。预训练模型只知道这些文本在语料中通常怎样接下去，并没有专门学过“按用户指定的格式、受众、长度和安全边界回答”。

这就是后训练的动机：把一个会续写文本的基础模型，进一步塑造成一个能响应开放式指令、遵守对话格式、偏向有用和安全输出的助手模型。

## Base model 和 instruction model 的区别

可以把两者的差别理解为目标函数和行为分布的差别。

Base model 学的是互联网和书面语料中的自然文本分布。它适合做续写、补全和少样本提示，但它不天然知道“用户请求”与“助手回答”之间的产品约定。

Instruction-tuned model 则经过后训练，见过大量“指令-回答”或“回答偏好”数据。它学到的是：输入是一条任务描述或对话上下文时，输出应该是有帮助、格式合适、尽量安全的回答。后训练不是简单增加参数，而是改变模型被优化的行为目标。

因此，规模提升和后训练解决的是两类问题：

- 扩大预训练规模主要提升知识覆盖、语言建模和上下文学习能力。
- 后训练主要提升指令遵循、对话可用性、偏好对齐和安全边界。

## 后训练的几种范式

现代大语言模型后训练常见四类方法。

| 方法 | 数据形式 | 作用 |
| --- | --- | --- |
| 监督微调 SFT | 指令 `x` 与人工或高质量回答 `y` | 让模型先学会基本指令格式和回答方式 |
| RLHF | 人类对多个回答的偏好排序 | 用人类偏好进一步优化回答质量 |
| RLAIF | 由另一个 AI 或规则系统给出偏好 | 降低人工偏好标注成本 |
| Reasoning RL / RLVR | 有可验证答案的问题与奖励 | 强化数学、代码、逻辑等可自动验证任务 |

SFT 通常是后训练的第一步。它把 base model 拉到“会回答指令”的区域。RLHF 或相关偏好优化方法再进一步把模型推向“人更喜欢的回答”。RLAIF 则把部分人类偏好标注替换成 AI 反馈。推理强化学习强调可验证答案，例如数学题、代码测试或形式化任务。

## InstructGPT 三步流水线

InstructGPT 是理解 ChatGPT 式后训练的经典例子。它把“让模型听指令”拆成三步：SFT、奖励建模、强化学习。

第一步是监督微调。收集一批 prompt，并让标注员写出理想回答。训练时把 prompt 和回答拼接给模型，但损失主要算在回答部分，让模型学习“看到这种指令时应该生成怎样的答复”。这一步成本高，因为完整答案需要人写，但它给模型建立了最基本的助手格式。

第二步是奖励建模。对同一个 prompt，让 SFT 模型生成多个候选回答，再请人类对这些回答排序。排序结果会变成偏好对：回答 `y_w` 比回答 `y_l` 更好。奖励模型 `r(x, y)` 的任务是给 prompt 和回答打一个标量分数，使得人类更偏好的回答得分更高。

常用的偏好模型可以写成：

```text
P(y_w 优于 y_l | x) = sigmoid(r(x, y_w) - r(x, y_l))
```

这个公式的直觉很简单：两个回答的分差越大，前者被认为更好的概率越高。相比让标注员给绝对分，两两比较更稳定，也更容易统一标准。

第三步是用奖励模型做强化学习。模型对 prompt 生成回答，奖励模型给回答打分，训练过程推动策略模型生成更高分的回答。InstructGPT 使用 PPO 作为优化算法，并加上 KL 惩罚，让新模型不要偏离 SFT 模型太远。这个约束很重要，因为只最大化奖励模型分数可能导致 reward hacking：模型学会钻奖励模型漏洞，而不是真的变得有帮助。

## KL 惩罚和对齐税

RLHF 的一个核心风险是过度优化奖励模型。奖励模型只是人类偏好的近似器，不是完美裁判。如果策略模型为了高分大幅偏离原来的语言分布，可能出现啰嗦、模式化、虚假自信，甚至专门利用奖励模型弱点的输出。

KL 惩罚的作用是把策略模型拉回参考模型附近：

```text
优化目标 = 奖励模型分数 - beta * KL(当前策略 || 参考策略)
```

这里的参考策略通常是 SFT 模型。`beta` 控制“追求高奖励”和“保持原行为分布”之间的平衡。

另一个问题叫对齐税：模型经过对齐后，在某些传统 NLP benchmark 上可能变差。原因是模型被调向了更符合人类偏好的助手行为，而不是单纯最大化原来任务格式下的分数。InstructGPT 使用 PPO-ptx 等方式混入预训练目标，试图减少这种能力损失。

## SFT 数据从哪里来

SFT 的关键不是只堆数量，而是覆盖足够多样的任务类型和回答风格。常见来源包括：

- 把现有 NLP 数据集改写成自然语言指令，例如 T0、FLAN。
- 在指令中加入少样本示例或思维链示范，例如 FLAN-T5。
- 用少量种子任务让模型自动扩展新指令和答案，例如 Self-Instruct。
- 用强模型生成回答，再训练较小开源模型模仿，例如 Alpaca。
- 使用真实用户分享的多轮对话，例如 Vicuna 使用的 ShareGPT 数据。
- 招募人工从零编写开放数据，例如 Dolly 和 OpenAssistant。
- 用少量但高质量的精选样本训练，例如 LIMA 强调“知识来自预训练，SFT 主要教输出格式”。

这些路线共同指向两个经验结论：质量通常比数量更重要，多样性通常比单一数据集规模更重要。SFT 无法把一个底座很弱的模型变成强模型，但能显著改变一个强 base model 的可用形态。

## RLHF 的角色分工

理解 RLHF 时，最好区分三类模型。

| 角色 | 含义 |
| --- | --- |
| Policy model | 当前正在被优化的语言模型，负责生成回答 |
| Reference model | 冻结的 SFT 模型，用于计算 KL 惩罚 |
| Reward model | 根据 prompt 和回答输出偏好分数的模型 |

Policy model 负责行动，reward model 负责评价，reference model 负责限制偏移。三者配合起来，才能让模型既向人类偏好靠近，又不因为过度优化奖励而跑偏。

## 复习时抓住什么

后训练这部分可以按一条主线记：

```text
Base model 会续写
-> SFT 教它按指令回答
-> RM 学人类偏好
-> RLHF 用偏好优化回答
-> KL / 预训练混合约束防止跑偏
```

几个容易混淆的点也要分开：

- SFT 数据是“指令-回答”对，RM 数据是“同一指令下哪个回答更好”的偏好排序。
- SFT 不需要强化学习，RLHF 才需要把模型当作策略来优化。
- 奖励模型不是最终要部署的助手，它是训练阶段的评分器。
- 后训练不是给模型增加知识的主要阶段，知识主要来自预训练；后训练主要决定模型如何选择和组织已有能力。

用一句话概括：后训练把语言模型从“会接文本”推向“能作为助手完成开放任务”，其中 SFT 建立指令格式，RLHF/RLAIF 引入偏好信号，而 KL 等约束负责防止模型为了高奖励偏离语言和安全边界。
