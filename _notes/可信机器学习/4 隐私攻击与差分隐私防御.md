> 这篇笔记整理可信机器学习中的隐私安全主线：模型为什么会泄露训练数据，成员推理、模型反演和训练数据提取分别在攻击什么，以及差分隐私、同态加密、安全多方计算和联邦学习各自解决哪一类问题。

## 为什么模型也会泄露隐私

传统数据库的隐私风险比较直观：如果攻击者拿到表，就能看到姓名、电话、地址、病历等字段。机器学习模型更隐蔽，因为它看起来只是参数或接口，但训练过程可能把数据中的个体信息压进了模型。

一个有用的判断是：

- 如果模型对训练样本明显比未见样本更自信，就可能暴露“某个人是否在训练集中”。
- 如果模型容量很大、数据有长尾样本，就可能记住罕见文本、邮箱、地址或其他高熵字符串。
- 如果接口返回过多置信度、梯度或中间表示，攻击者就有更多线索做推理。

所以“模型不等于原始数据”并不意味着模型没有隐私风险。对过拟合模型来说，参数和输出都可能成为训练数据的侧信道。

## 三类常见隐私攻击

隐私攻击的共同目标是从模型中恢复本不应该公开的信息。可以按攻击结果分成三类。

### 成员推理攻击

成员推理攻击（Membership Inference Attack, MIA）要回答一个二分类问题：

```text
给定样本 x 和目标模型 f，
判断 x 是否属于 f 的训练集。
```

最基本的信号是置信度差异。模型通常在训练样本上损失更低、softmax 更尖锐；在未见样本上更不确定。攻击者可以把模型输出向量喂给一个二分类器，预测 `in` 或 `out`。

Shokri 等人的影子训练方法把这个思路系统化：

1. 构造若干份与目标任务分布相近的影子数据集。
2. 训练影子模型，模拟目标模型的行为。
3. 对影子模型的训练样本和测试样本收集输出向量，并标注 `in` / `out`。
4. 用这些样本训练攻击模型，再迁移到目标模型上。

这类攻击的关键不是“模型说错了”，而是“模型对见过的样本表现得太不一样”。泛化间隙越大，成员推理越容易成功。

### 模型反演与特征重建

模型反演（Model Inversion）关心的是：能否从模型输出中重建输入的某些属性，甚至重建某类样本的代表形态。

以人脸识别为例，攻击者固定目标类别 `y`，在输入空间里优化一张图像，使模型对 `y` 的置信度最大：

```text
maximize   f(x)[y] - regularization(x)
subject to x is a valid input
```

正则项用于避免得到纯粹的噪声或对抗样本。最后得到的图像可能不是某个训练样本的逐像素复刻，但仍可能暴露该类别的敏感特征。

模型反演的风险取决于任务和数据。如果类别本身对应具体个人、疾病状态或身份属性，那么“类平均特征”也可能是隐私泄漏。

### 训练数据提取

训练数据提取（Training Data Extraction）比模型反演更强：攻击者希望拿到模型逐字记住的训练片段。

大语言模型中，这个问题尤其突出。Carlini 等人的 GPT-2 提取实验展示了一个典型流程：

1. 从模型中生成大量候选文本。
2. 用困惑度、zlib 压缩长度、大小写敏感度、小模型对比等指标排序。
3. 人工或程序核验候选是否来自训练语料。

核心信号是“低困惑度 + 高字符串复杂度”。如果一段邮箱、电话号码、UUID 或地址本身很难压缩，但大模型却异常自信地续写出来，就很可能是字面记忆。

这说明模型规模、数据重复、长尾文本和训练轮数都会影响隐私风险。模型越能记住稀有样本，越需要认真处理训练数据治理与输出过滤。

## 泄漏的根因

隐私泄漏通常来自以下几类机制。

**过拟合。** 训练损失远低于测试损失时，模型已经学会区分训练样本和未见样本。成员推理攻击正是利用这个差异。

**高容量。** 参数量越大，模型越有能力记住长尾样本。大模型的泛化能力和记忆能力并不矛盾。

**数据长尾。** 罕见字符串、少数类样本、异常医疗记录、独特行为轨迹都更容易被模型记住。

**接口暴露过多。** 返回完整置信度、logit、梯度或可重复采样结果，会放大攻击者的信息量。

**缺少数据最小化。** 如果训练集中保留了任务不需要的个人信息，那么模型就有机会学到并泄露这些信息。

## 防御全景

隐私防御不是单一技术，而是按威胁位置分层处理。

| 技术 | 主要保护对象 | 直觉 | 代价 |
|------|--------------|------|------|
| 数据最小化 | 原始训练数据 | 不收集、不训练不必要信息 | 需要前期数据治理 |
| 同态加密 | 计算过程中的明文 | 在密文上计算 | 计算和存储开销大 |
| 安全多方计算 | 多方输入 | 多方协作计算但不暴露各自数据 | 通信开销大 |
| 联邦学习 | 原始数据位置 | 数据留在本地，只上传模型更新 | 梯度仍可能泄露 |
| 差分隐私 | 单条记录影响 | 输出对任一个体不敏感 | 精度下降，需要隐私预算管理 |

这些方法解决的问题不同。同态加密和安全多方计算更偏“计算过程看不到明文”；联邦学习更偏“数据不离开本地”；差分隐私更偏“即使看到输出，也难判断某个人是否参与训练”。

## 差分隐私的核心定义

差分隐私（Differential Privacy, DP）给出一种形式化保证：如果两个数据集只差一条记录，那么算法在这两个数据集上的输出分布应该几乎不可区分。

设相邻数据集 `D` 和 `D'` 只差一条记录。随机算法 `M` 满足 `(epsilon, delta)`-DP，当且仅当对任意输出集合 `S`：

```text
Pr[M(D) in S] <= exp(epsilon) * Pr[M(D') in S] + delta
```

其中：

- `epsilon` 是隐私预算，越小保护越强。
- `delta` 是小概率失败项，通常应远小于 `1 / n`。
- 随机性是必要的，因为完全确定的精确回答通常会暴露单条记录的影响。

一个简单例子是计数查询。如果数据库里“单身人数”因为张三加入而从 2 变成 3，那么精确回答会直接泄露张三状态。DP 的做法是给结果加噪声，使攻击者无法可靠区分“张三在库中”和“张三不在库中”。

## DP-SGD 如何保护神经网络训练

神经网络训练中常用 DP-SGD。它把普通 SGD 改成三步：

```text
for each batch:
    compute per-sample gradient g_i
    clip each g_i to norm C
    average clipped gradients and add Gaussian noise
    update model parameters
```

逐样本梯度裁剪限制了单个样本对更新方向的最大影响；高斯噪声让攻击者难以从最终模型或中间更新中反推出某条记录是否参与训练。

DP-SGD 的代价也很明确：

- 噪声会降低精度，尤其在小数据集或复杂任务上更明显。
- 需要记录隐私预算，训练轮数越多通常消耗越多。
- 逐样本梯度计算比普通 batch 梯度更贵。

因此 DP-SGD 不是“免费增强”，而是在隐私风险和模型性能之间做可审计的权衡。

## 与可信机器学习其他主题的关系

隐私章节和前面的对抗鲁棒性、后门攻击并不是孤立的。

- 对抗样本强调测试时扰动会改变预测。
- 后门攻击强调训练阶段可以植入隐藏触发器。
- 隐私攻击强调模型输出会泄露训练数据。

三者都说明：只看测试准确率不足以判断模型是否可信。一个模型可以准确率很高，却对小扰动敏感；也可以干净准确率正常，却含有后门；还可以任务表现很好，却记住了训练数据中的私人信息。

复习时可以把可信机器学习理解成四个问题：

1. 模型面对输入扰动是否稳定？
2. 模型训练链路是否被污染？
3. 模型决策是否可解释、可审计？
4. 模型是否泄露训练数据或用户信息？

隐私部分回答的是第四个问题。

## 复习抓手

这章最容易混淆的是攻击目标。可以按下面方式记：

- MIA：判断样本“是否在训练集”。
- Model Inversion：恢复类别或样本的敏感特征。
- Extraction：提取模型逐字记住的训练内容。
- HE / SMPC：让计算时看不到明文。
- FL：让原始数据尽量留在本地。
- DP：让输出对任意单条记录不敏感。

真正做系统设计时，通常需要组合这些方法：先减少训练数据中的敏感字段，再限制接口输出，必要时使用 DP 训练或 DP 统计，并对高风险模型做成员推理和数据提取测试。
