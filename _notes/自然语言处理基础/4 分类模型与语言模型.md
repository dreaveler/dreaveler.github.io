> 北京大学《自然语言处理基础》第四讲整理。内容基于课程 PDF《More About Classification》与《Language Models》，适合作为从分类任务过渡到语言建模的复习入口。

## 本讲在课程里的位置

这一讲把前面偏任务定义和问题背景的内容，往“怎么建模、怎么评测、怎么给句子赋概率”推进了一步。可以分成两条主线：

1. 分类模型怎么看待输入和标签
2. 语言模型如何给一个词序列分配概率

## 1. 生成式模型与判别式模型

课程里先从分类问题出发。目标可以写成：

```text
y = g(x) = argmax p(y|x)
```

围绕这个目标，有两种典型建模视角：

| 视角 | 学习对象 | 直觉 |
| --- | --- | --- |
| 判别式模型 | `p(y|x)` | 直接学习“给定输入后属于哪个类别” |
| 生成式模型 | `p(x, y)` 或 `p(x|y)p(y)` | 先学“每一类样本长什么样”，再比较哪一类更可能生成当前输入 |

一个好记的理解方式是：

- 判别式模型更关心分类边界
- 生成式模型更关心数据生成过程

在 NLP 里，这种区分很重要，因为后面的朴素贝叶斯、HMM、语言模型等内容，都带有明显的生成式味道；而逻辑回归、最大熵模型、很多神经分类器，则更接近判别式路线。

## 2. 分类任务的评测

只看“预测对了多少”往往不够，尤其是在类别不平衡时。因此这讲还强调了分类评测指标。

### Accuracy

```text
Accuracy = 正确预测样本数 / 总样本数
```

它适合类分布比较均衡的任务，但在正负样本极不均衡时可能失真。

### Precision 与 Recall

- Precision：预测为正的样本里，有多少是真的正例
- Recall：真实正例里，有多少被模型找出来

很多信息抽取、检索、命名实体识别任务，都会同时关心这两个指标。

### F1

F1 是 Precision 和 Recall 的调和平均：

```text
F1 = 2PR / (P + R)
```

复习时要记住：

- 如果更怕误报，重点看 Precision
- 如果更怕漏报，重点看 Recall
- 想综合比较时，常看 F1

## 3. 什么是语言模型

语言模型的核心任务是：

给一个词序列分配概率。

如果把所有合法句子组成集合 `S`，那么语言模型就是定义在 `S` 上的一个概率分布 `p(s)`。直观上：

- 自然、通顺、符合语言习惯的句子，概率应更高
- 词序混乱、结构异常的句子，概率应更低

例如：

- `the dog barks .` 的概率应较大
- `dog the bark a laugh smile .` 的概率应极小

这个定义说明，语言模型本质上是在量化一句话的“流畅性”和“自然程度”。

## 4. 为什么需要语言模型

语言模型并不只是为“生成句子”服务，它在很多任务中都充当打分器。

典型场景包括：

- 语音识别
- OCR 与手写识别
- 分词
- 机器翻译
- 文本生成
- 问答与对话

一个经典例子是语音识别中的歧义：

- `recognize speech`
- `wreck a nice beach`

如果只看声学信号，两者可能都接近；这时语言模型会偏向更自然、更合理的词序列。因此可以把总体目标理解为：

```text
最优输出 = 信号匹配分数 + 语言流畅度分数
```

## 5. 链式法则与 N-gram 模型

对句子 `w1, w2, ..., wn`，联合概率可以用链式法则展开：

```text
p(w1, w2, ..., wn) = p(w1) p(w2|w1) p(w3|w1,w2) ... p(wn|w1,...,w(n-1))
```

问题在于，真实条件历史太长，直接估计几乎不可行。于是引入马尔可夫假设：

当前词只依赖有限个前文词。

这就得到 N-gram 语言模型：

- Unigram：只看当前词本身
- Bigram：只看前 1 个词
- Trigram：只看前 2 个词
- 一般 N-gram：只看前 `N-1` 个词

对应公式可以写成：

```text
p(wi | w1, ..., w(i-1)) ≈ p(wi | w(i-N+1), ..., w(i-1))
```

## 6. 参数估计

N-gram 最直接的做法是最大似然估计，也就是用频数近似概率。

例如 bigram 的估计：

```text
p(wi | w(i-1)) = count(w(i-1), wi) / count(w(i-1))
```

这个思路简单直观，但马上会遇到一个核心问题：

训练语料里没出现过的组合，概率会被估成 0。

而自然语言的长尾现象非常明显，所以“未见事件”不是偶然，而是常态。

## 7. 平滑与插值

为了解决未见事件问题，课程里强调了两类思路：

### 线性插值

把不同阶的模型混合起来：

```text
p = λ1 p_unigram + λ2 p_bigram + λ3 p_trigram
```

直觉是：

- 高阶模型更具体，但更稀疏
- 低阶模型更粗糙，但更稳定

插值是在“利用上下文”与“避免过拟合稀疏计数”之间折中。

### 平滑

平滑的目标是把一部分概率质量分给未见事件，避免 0 概率。

复习时最重要的不是背具体公式，而是记住三件事：

1. 为什么需要平滑：自然语言存在大量长尾与未见组合
2. 平滑在做什么：重新分配概率质量
3. 平滑的代价是什么：已见事件概率会被适度压低

## 8. 语言模型的评估

课程里评估语言模型时，重点不是看分类准确率，而是看模型对真实语料的概率分配质量。

常见指标包括：

### 交叉熵

如果模型对真实测试语料赋的概率越高，交叉熵就越低，说明模型越好。

### 困惑度 Perplexity

困惑度可以理解成模型对下一个词有多“困惑”。

- 困惑度越低越好
- 如果模型总把合理下一个词排在高概率位置，困惑度就会下降

复习时可以把它记成：

语言模型好不好，本质上看它是否能把真实句子判得更“自然”、更“高概率”。

## 9. 本讲复习抓手

如果要把这一讲压缩成最少的骨架，可以记下面几条：

- 分类模型有生成式与判别式两种视角
- 分类评测常看 Accuracy、Precision、Recall、F1
- 语言模型的任务是给句子赋概率
- N-gram 来自链式法则加马尔可夫假设
- 未见事件会导致 0 概率，因此需要平滑和插值
- 语言模型评估重点看交叉熵和困惑度

把这些主线记住，后面再学神经语言模型、预训练模型和 Transformer 时，会更容易看清它们到底是在替换哪一层旧方法。
