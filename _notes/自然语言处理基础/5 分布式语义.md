> 北京大学《自然语言处理基础》第 5 讲整理。内容基于课程讲授、课件与已整理复习笔记，主线是从共现矩阵、PPMI、SVD 一路走到 word embedding。

## 核心问题

如何让计算机表示“一个词的含义”？

最直接的 one-hot 表示虽然简单，但有两个明显问题：

- 任意两个词几乎都彼此正交，无法表达相似性
- 维度等于词表大小，既高维又稀疏

分布式语义的基本思路是：

如果两个词出现在相似上下文里，那么它们的含义通常也相近。

这就是经典的 distributional hypothesis。

## 从共现出发

最早的一类方法不是直接学神经网络，而是先统计“词和上下文一起出现了多少次”。

常见两种矩阵：

### 1. term-context 矩阵

矩阵中的每一行对应一个词，每一列对应一个上下文词，元素表示它们在固定窗口中共现的次数。

这种矩阵更适合做词义相似度分析。

### 2. term-document 矩阵

元素表示某个词在某篇文档中出现多少次。

这种矩阵更常见于检索与 LSA。

## 为什么不能直接用原始词频

原始计数有一个问题：高频词会主导结果。

像“的”“是”“and”这类词会在几乎任何上下文里出现，它们的高频并不意味着有强语义关联。

因此通常要先做加权。

## PMI 与 PPMI

衡量“词和上下文是否比独立情形更容易一起出现”的典型方法是 PMI：

```text
PMI(w, c) = log [ P(w, c) / (P(w) P(c)) ]
```

直觉：

- 若 `w` 与 `c` 独立，则 PMI 为 0
- 若它们更倾向于一起出现，则 PMI 为正
- 若它们比独立情况下更少一起出现，则 PMI 为负

实践中更常用 PPMI：

```text
PPMI(w, c) = max(PMI(w, c), 0)
```

原因是负 PMI 在真实语料里往往噪声较大，而我们更关心“强共现”的正信号。

## 向量相似度

当每个词都被表示成一个向量后，就需要比较两个词是否相似。

最常用的是余弦相似度：

```text
cos(u, v) = (u · v) / (||u|| ||v||)
```

它比较的是方向而不是长度，因此能减弱词频大小带来的影响。

## 从稀疏到稠密：SVD 与 LSA

即使做了 PPMI，加权后的矩阵仍然很大、很稀疏。于是下一步通常是降维。

最经典的方法是 SVD：

```text
A ≈ Uk Sk Vk^T
```

只保留最大的前 `k` 个奇异值，就能得到一个低秩近似。

在 NLP 里，这带来两个直接收益：

- 把高维稀疏表示压成低维稠密表示
- 把一些局部噪声平滑掉，让语义关系更稳定

LSA 的核心就是：先构造词-文档或词-上下文矩阵，再用 SVD 得到低维语义空间。

## count-based 与 prediction-based

课程里很重要的一条线索，是区分两大类词表示方法：

### 1. count-based

先统计共现，再做权重和降维。

代表：

- 共现矩阵
- PPMI
- TF-IDF
- LSA / SVD

### 2. prediction-based

不显式存整张大矩阵，而是直接通过预测任务学习向量。

代表：

- word2vec
- GloVe
- 神经语言模型

count-based 更容易解释，prediction-based 通常效果更强、扩展性更好。

## word2vec 的直觉

word2vec 的核心不是“记住词”，而是在训练中逼迫模型学会：

- 给定上下文预测中心词，或
- 给定中心词预测上下文

两种典型结构：

- CBOW：用上下文预测中心词
- Skip-gram：用中心词预测上下文

训练完成后，中间隐藏层里的参数就变成了词向量。

## 为什么 word embedding 比 one-hot 更好

因为 embedding 把“语义相近”编码成“向量相近”。

这意味着：

- 近义词会落在空间里相近的位置
- 模型能把低频词和高频词之间的统计规律部分共享
- 下游任务可以直接利用这些连续特征

这是从离散符号走向连续表示的一大步。

## SGNS 和 PMI 的联系

课程复习里一个很值得记的点是：

Skip-gram with Negative Sampling 并不只是一个“经验上好用的神经方法”，它和 PMI 矩阵分解在数学上是有关联的。

可以把它粗略理解成：

SGNS 在隐式地逼近一个“平移后的 PMI 矩阵”。

这个结论的重要性在于，它把 count-based 和 prediction-based 两条路线真正连起来了：

- 它们不是完全割裂的两种世界
- 很多神经词向量方法，本质上也可以理解成一种更高效的矩阵分解

## 常见评估方式

### 内在评估

直接看向量本身的性质，例如：

- 词相似度任务
- 类比任务
- 最近邻是否合理

### 外在评估

把词向量放进下游任务里，例如：

- 文本分类
- 序列标注
- 机器翻译

最终看下游指标是否提升。

一般来说，外在评估更接近“真正有没有用”。

## 这一讲的主线总结

可以把整讲压缩成下面这条链：

1. 词义可以由上下文分布来近似表示
2. 先统计共现关系，得到高维稀疏矩阵
3. 用 PMI / PPMI 或 TF-IDF 做加权
4. 用余弦相似度比较词语关系
5. 用 SVD / LSA 压成低维稠密表示
6. 进一步发展到 word2vec 等预测式 embedding

## 复习抓手

考前如果只想抓最核心的点，优先记住：

- distributional hypothesis 是整章出发点
- PPMI 解决的是“共现强度如何合理加权”
- 余弦相似度用来比较词向量方向
- SVD / LSA 是从稀疏统计表示走向稠密表示的关键一步
- word2vec 属于 prediction-based embedding
- SGNS 与 PMI 分解有理论联系
