> 这篇笔记整理自然语言处理基础中“大语言模型训练数据”的内容。它不讨论 Transformer 结构本身，而是回答另一个同样重要的问题：预训练语料从哪里来，怎样清洗，为什么数据质量会直接影响模型能力。

## 为什么要单独讨论数据

语言模型的训练目标通常是“根据已有上下文预测下一个 token”。这意味着模型学到的知识、表达方式和推理模式，最终都来自它在预训练阶段看过的数据。模型结构决定了它怎样学习，数据则决定了它能学到什么。

早期预训练模型常用 Wikipedia、BooksCorpus 等相对干净的小规模文本。到了 GPT-2、GPT-3 和后来的开源大模型时代，训练语料扩展到网页、书籍、论文、代码、论坛、百科和专业领域资料。数据规模从几十亿词增长到数万亿 token，数据工程也从“收集语料”变成了“从巨大脏语料中筛出高质量文本”。

一个重要背景是：很多开权重模型并不公开完整训练数据配方。模型权重可以下载，但数据来源、比例、清洗规则、过滤阈值常常是保密的。因此，开源社区也在复现和公开数据构建流程，例如 C4、The Pile、Dolma、FineWeb 和 DCLM。

## 五个基础概念

理解大模型数据工程前，先明确几个词：

| 概念 | 含义 |
| --- | --- |
| Token | 模型处理文本的基本单位，可以是词、子词、字符或汉字片段。训练规模常用 token 数衡量。 |
| Corpus | 语料集合，也就是训练模型时喂给它的一批文本。 |
| Common Crawl | 一个持续抓取互联网网页并公开数据的项目，是许多预训练数据集的原料来源。 |
| Scraping | 自动抓取网页内容。抓到的通常是 HTML、菜单、广告、脚本和正文混在一起的脏数据。 |
| Deduplication | 去重，包括删除完全重复和近似重复的文本，避免模型反复学习同一内容。 |

这些概念串起来就是：从互联网等来源抓取大量文本，把它们转换成模型可读的 token 序列，再清洗、过滤、去重，最后形成预训练语料。

## 预训练数据的演进

GPT-2 之前，预训练数据以精选小语料为主。例如 BERT 使用 BooksCorpus 和英文 Wikipedia。这类语料干净、结构好，但规模和覆盖面有限。

GPT-2 的 WebText 是一个转折点。它没有直接抓全网，而是抓取 Reddit 上被用户分享且获得一定点赞的外链页面。这里的思想是：用人类社区的行为信号近似判断网页质量。相比纯随机网页，WebText 更可能包含有价值、可读、主题多样的自然语言文本。

再往后，Common Crawl 成为主要原料。它规模巨大，但非常脏：网页模板、菜单、脚本提示、广告、乱码、重复段落都大量存在。直接用 Common Crawl 训练模型会浪费算力，并可能让模型学到低质量模式。所以现代数据集的关键不只是“抓得多”，而是“洗得好”。

C4 是一个经典清洗方案。它从 Common Crawl 中用一套启发式规则过滤文本，例如只保留像完整句子的行、删除过短页面、删除 JavaScript 提示、删除 lorem ipsum 占位文本、过滤脏词、删除隐私政策和 cookie 提示等样板内容，并做语言识别和去重。C4 的意义在于，它展示了大规模网页语料可以通过明确规则变成相对干净的预训练数据。

GPT-3 进一步引入模型法过滤：用高质量语料作为正例、原始 Common Crawl 作为负例，训练分类器给网页打质量分，再根据分数筛选。它还会对 Wikipedia、书籍等高质量来源上采样，让模型在同样训练预算内更多看到可靠文本。

## 开源数据集的几条主线

后来的开源数据集大致沿着三条路线发展。

第一条是扩大规模。The Pile、RedPajama、Dolma、FineWeb、DCLM 等数据集把公开预训练语料推进到数千亿乃至数万亿 token。规模扩大本身不是目的，目标是在更大覆盖面中筛出足够多的高质量文本。

第二条是增加多样性。The Pile 混合了网页、论文、医学文献、代码、书籍、百科和问答社区。这样做是为了让模型学习不同文体、领域知识和任务格式，而不是只会生成普通网页文本。

第三条是做领域专门化。代码数据有 The Stack，数学数据有 OpenWebMath / FineMath，中文和多语数据也有专门构建的集合。因为通用网页语料对某些能力支持不足，专门领域需要额外补强。

这些路线共同说明：训练数据并不是一个简单的“大文件”，而是一套关于来源、比例、清洗、去重、采样和合规性的工程决策。

## 数据规模和模型规模的关系

缩放律说明，模型性能会随参数量、数据量和算力增加而规律性提升。但如果只增加参数而不给足够数据，模型会“吃不饱”；如果数据足够多，较小模型也可能获得很强能力。

Chinchilla 系列工作强调，在固定训练算力下，过去很多模型参数过大、训练 token 不足。之后的趋势是：模型参数不一定无限增大，但训练 token 数快速上升。Llama 3、Qwen、OLMo 等模型都体现了“中等或较大参数 + 海量高质量数据”的路线。

因此，大模型竞争不只是模型结构竞争，也是数据配方竞争。一个更干净、更覆盖关键领域、更少重复的数据集，可能比单纯更大的脏数据更有效。

## 数据治理流水线

现代预训练数据通常可以拆成三步：

```text
Acquisition -> Linearization -> Filtering
获取        -> 线性化        -> 过滤
```

获取决定从哪些来源拿数据。Common Crawl 通常是底座，但还会补充 Wikipedia、书籍、论文、代码、数学网页、问答社区等来源。补充来源的作用是弥补网页语料的覆盖缺口，并提高知识密度。

线性化把 HTML、PDF、Word、LaTeX、代码仓库等不同格式转换成纯文本。这个步骤看似机械，实际非常重要。网页正文抽取如果做得差，模型会学到导航栏、脚本、版权声明和广告文本；PDF 抽取如果做得差，会产生错乱的换行、页眉页脚和公式碎片。

过滤负责删掉低质量、有害或重复内容。常见过滤方式包括：

- 精确去重：删除完全相同的文档或段落。
- 模糊去重：删除高度相似的文档，常用 MinHash、后缀数组等方法。
- 启发式过滤：用长度、标点、语言、脏词、模板词等规则筛掉明显低质文本。
- 模型法过滤：训练分类器或使用大模型判断文本质量，再按分数筛选。

一个常见顺序是先用便宜的启发式规则删掉明显垃圾，再做去重，最后用模型法过滤更细粒度地区分质量。这样可以避免在大量垃圾文本上浪费昂贵的模型判断。

## 线性化为什么重要

线性化的目标是把“人看到的内容”尽量还原成“模型应当学习的文本”。以网页为例，原始 HTML 里有正文、标题、链接、脚本、样式、广告和评论区。模型需要的是有语义的正文，而不是所有标签和模板。

Common Crawl 中常见的 WARC 文件保存的是原始抓取结果，信息完整但非常脏；WET 文件是抽取出的纯文本，更方便训练但可能已经丢失结构或引入抽取错误。数据构建者需要在保留信息和减少噪声之间权衡。

PDF 和论文也类似。论文中的公式、表格、引用、脚注和双栏布局都可能让抽取结果变乱。如果线性化质量差，即使原始来源很高质量，最终训练文本也会变成低质量噪声。

## 去重不仅是节省空间

去重的作用不只是减少数据量。重复文本会带来几个问题：

- 浪费训练预算，让模型反复看同一内容。
- 放大某些网站、模板或观点的权重。
- 增加训练集和评测集重叠的风险，使评测结果虚高。
- 让模型更容易记忆具体文本，增加隐私和版权风险。

精确去重能处理完全重复，模糊去重能处理轻微改写、转载和模板化页面。对于网页语料，模糊去重尤其重要，因为很多网站会转载同一篇文章，或者在相同模板中嵌入少量不同正文。

## 启发式过滤和模型法过滤

启发式过滤的优点是便宜、可解释、容易大规模运行。缺点是规则粗糙，可能误删有价值文本，也可能漏掉复杂噪声。C4 的规则就是典型例子：它能快速删掉大量明显垃圾，但无法理解文本是否真正有知识价值。

模型法过滤更灵活。可以用高质量数据作为正例、低质量网页作为负例，训练一个分类器判断新文档是否像高质量文本。也可以用大模型标注小批量数据，再蒸馏成便宜分类器用于大规模过滤。

模型法过滤的风险是偏见会被放大。如果正例只来自某些文体或领域，分类器可能把其他有价值文本误判为低质量。因此实践中常常不会只保留最高分文本，而是在质量和多样性之间设定折中。

## 数据合规和版权问题

预训练数据还涉及版权、隐私和使用许可。网页公开可访问不等于一定可以无条件用于训练。代码、书籍、论文、论坛回答、个人信息都可能有不同的许可和风险。

开源模型若希望被广泛使用，数据来源通常要尽量选择公开可得、许可清晰、隐私风险较低的材料。LLaMA 等模型报告中强调“不使用产品私有数据”或“移除个人隐私信息”，正是因为数据治理已经不仅是技术问题，也涉及法律和伦理边界。

## 复习抓手

这部分内容可以按三层来记：

- 历史层：从 Wikipedia / BooksCorpus，到 WebText，再到 Common Crawl、C4、GPT-3 分类器过滤和现代开源数据集。
- 流水线层：获取、线性化、过滤。每一步都可能决定最终语料质量。
- 方法层：去重、启发式过滤、模型法过滤、上采样和领域补充。

最重要的结论是：大语言模型的数据工程不是简单堆 token。真正关键的是用有限训练预算让模型尽可能多地看到干净、多样、可靠、覆盖关键领域的文本。
