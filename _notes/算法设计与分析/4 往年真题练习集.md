## 1. 递推方程求解

### 问题 1.1（2015 年，10 分）

**求解递推方程。要求给出求解过程。**

已知递推方程：

$$
T(n) = 4T(n/2) + n^2, \quad T(1) = 1
$$

<details>
<summary>查看答案</summary>

**答案：** `Theta(n^2 log n)`

**解析：**
根据主定理第二种情况，`a = 4, b = 2, f(n) = n^2`。

因为 `n^(log_b a) = n^(log_2 4) = n^2`，所以 `f(n)` 与递归主项同阶，因此

$$
T(n) = \Theta(n^2 \log n)
$$

</details>

### 问题 1.2（2015 年，10 分）

**求解递推方程。要求给出求解过程。**

已知递推方程：

$$
T(n) = 2T(n/2) + n\log n, \quad T(1) = 1
$$

<details>
<summary>查看答案</summary>

**答案：** `Theta(n log^2 n)`

**解析：**
这一题不能直接套主定理第二种或第三种情形，更稳妥的做法是用递归树。

展开后每层总代价大致是 `n(log n - k)`，层数为 `log n`，因此总和为

$$
\Theta(n \log^2 n)
$$

</details>

### 问题 1.3（2017 年，10 分）

**求解递推方程。要求给出求解过程。**

1. `T(n) = T(n-1) + \log(5^n), \quad T(1) = 1`
2. `T(n) = 2T(n/2) + n^2\log n, \quad T(1) = 1`

<details>
<summary>查看答案</summary>

**答案：**

1. `Theta(n^2)`
2. `Theta(n^2 log n)`

**解析：**

1. `\log(5^n) = n\log 5`，因此递推可看成 `T(n) = T(n-1) + Theta(n)`，累加后得到 `Theta(n^2)`。
2. `a = 2, b = 2`，故 `n^(log_b a) = n`，而 `f(n) = n^2 log n` 多项式地更大并满足正则条件，所以结果为 `Theta(n^2 log n)`。

</details>

### 问题 1.4（2019 年期中，10 分）

**求解递推方程。每题 5 分。**

1. `T(n) = T(n-1) + \log n, \quad T(1) = 1`
2. `T(n) = 4T(n/2) + n^2, \quad T(1) = 1`

<details>
<summary>查看答案</summary>

**答案：**

1. `Theta(n log n)`
2. `Theta(n^2 log n)`

**解析：**

1. 第一题可以从 `\sum_{k=1}^n \log k = \log(n!)` 入手，再由 Stirling 量级得到 `Theta(n log n)`。
2. 第二题是标准主定理第二种情况。

</details>

### 问题 1.5（2021 年，10 分）

**从下述三个算法中选择一个最坏情况下时间复杂度最低的算法，简要说明理由。**

- 算法 A：把原问题划分成 6 个规模减半的子问题，线性时间合并
- 算法 B：递归求解 2 个规模为 `n-1` 的子问题，常数时间合并
- 算法 C：把原问题划分成 9 个规模为 `n/3` 的子问题，线性时间合并

<details>
<summary>查看答案</summary>

**答案：** 选择算法 A。

**解析：**

- A: `T_A(n) = 6T(n/2) + O(n) = Theta(n^(log_2 6))`
- B: `T_B(n) = 2T(n-1) + O(1) = Theta(2^n)`
- C: `T_C(n) = 9T(n/3) + O(n) = Theta(n^2)`

因为 `log_2 6 ≈ 2.58`，所以增长速度满足 `A < C < B`，最优的是 A。

</details>

### 问题 1.6（2022 年，10 分）

**求解递推方程，要求给出求解过程。**

1. `T(n) = 9T(n/3) + n, \quad T(1) = 1`
2. `T(n) = T(n-1) + \log 3^n, \quad T(1) = 1`

<details>
<summary>查看答案</summary>

**答案：**

1. `Theta(n^2)`
2. `Theta(n^2)`

**解析：**

1. 第一题直接用主定理，`a = 9, b = 3`，得到 `n^(log_3 9) = n^2`。
2. 第二题中 `\log 3^n = n \log 3`，所以本质上仍是线性递增加和，最终得到平方量级。

</details>

### 问题 1.7（2023 年，10 分）

**求解递推方程，要求给出求解过程。**

1. `T(n) = 4T(n/2) + n^2, \quad T(1) = 1`
2. `T(n) = 2T(n/2) + n^2\log n, \quad T(1) = 1`

<details>
<summary>查看答案</summary>

**答案：**

1. `Theta(n^2 log n)`
2. `Theta(n^2 log n)`

**解析：**

1. 第一题仍是主定理第二种情况，不要误判成 `Theta(n^2)`。
2. 第二题里 `f(n)` 显著大于 `n^(log_2 2)=n`，所以由非递归项主导，结论为 `Theta(n^2 log n)`。

</details>

## 2. 分治算法设计

### 问题 2.1（2018 年，10 分）

**下面是算法 `Find` 的伪码。输入 `S` 是 `n` 个互不相等的数的数组。请问该算法输出什么？该算法在最坏情况下执行了多少次比较？**

```text
Find(S, n)
1. if S[1] > S[2]
2.    then x <- S[1]; y <- S[2]
3.    else y <- S[1]; x <- S[2]
4. for i <- 3 to n do
5.    if S[i] < y
6.       then if S[i] < x
7.          then y <- x; x <- S[i]
8.          else y <- S[i]
9. return y
```

<details>
<summary>查看答案</summary>

**答案：** 输出数组中的第二小元素。

**解析：**

- `x` 始终维护当前最小值
- `y` 始终维护当前第二小值
- 第 1 行比较 1 次
- 第 5、6 行在最坏情况下各执行 `n-2` 次

因此总比较次数为：

```text
1 + (n - 2) + (n - 2) = 2n - 3
```

</details>

## 3. 动态规划

### 做这类题时先检查四件事

往年题刷下来，动态规划最容易错的不是算不出，而是前置结构没写清楚。建议固定按下面顺序检查：

1. 状态是什么
2. 转移从哪里来
3. 边界条件是什么
4. 是只求最优值，还是还要恢复方案

如果题目是区间型、前缀型、容量型、阶段型，这个顺序尤其稳。

## 4. 贪心法

### 刷题时不要只盯策略

往年题里，贪心题经常不是考“能不能想到按什么排序”，而是考你能不能补上正确性证明。最常用的两种证明方式：

- 交换论证
- 归纳法

做题时至少要把这三点写出来：

1. 贪心规则是什么
2. 为什么局部选择安全
3. 复杂度是多少

## 5. 回溯、分支限界与平摊分析

### 最后几类题更像“模板应用题”

这几类题往往分值不低，但结构相对固定：

- 回溯：写解向量、搜索树、约束条件、剪枝
- 分支限界：在回溯基础上补上界或下界
- 平摊分析：说明用聚合、银行家还是势能法，并把每类操作的平摊代价算清楚

如果只是凭印象写结论，很容易丢掉过程分。
