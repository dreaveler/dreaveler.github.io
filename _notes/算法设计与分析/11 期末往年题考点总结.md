> 这篇笔记整理《算法设计与分析》期末复习的题型地图。内容基于 2009-2022 年期末卷与整理稿，目标不是背题，而是把高频模块、答题格式和典型证明套路放到一张能直接用于考前自查的表里。

## 期末卷到底在考什么

这门课的期末卷通常由客观题和算法设计大题组成。客观题覆盖面广，主要考概念辨析；大题则反复落在若干稳定模块上：

- 渐进分析与递归方程
- 分治法、比较下界与选择问题
- 动态规划
- 贪心、回溯与分支限界
- 图算法、网络流与线性规划建模
- 摊还分析
- 随机算法
- 近似算法
- NP 完全性证明

其中 NP 完全性、递归方程、网络流、摊还分析、随机算法、近似算法最值得优先复习。它们一方面出现频率高，另一方面答题模板比较固定，认真整理以后最容易把分拿稳。

## 答题格式比想法更重要

很多算法题不是只写出一个想法就结束，而是要求把算法设计过程写完整。可以按下面模板检查：

1. 先用自然语言说明算法思想。
2. 给出关键状态、数据结构或构造方式。
3. 写出递推、约束、增广路径、归约映射等核心形式化内容。
4. 证明正确性。
5. 分析时间复杂度。

动态规划题尤其要写齐四件事：状态定义、递推方程、边界条件、标记函数。贪心题必须补正确性证明，常用交换论证或“贪心选择可以延拓为最优解”的结构。NP 完全性题必须先证明问题在 NP 中，再从一个已知 NP 完全问题做多项式归约。

## 递归方程与分治

递归方程是期末中最稳定的基础题。常见工具有主定理、递归树、代入法和变量替换。

遇到

```text
T(n) = aT(n/b) + f(n)
```

先和临界项 `n^(log_b a)` 比较。如果 `f(n)` 更小，复杂度由叶子层主导；如果同阶，多一个 `log n`；如果更大并满足正则条件，复杂度由根层主导。

更容易出错的是带变量替换的递归。例如：

```text
T(n) = T(sqrt(n)) + 1
```

令 `n = 2^m`，就得到 `S(m) = S(m/2) + 1`，所以 `T(n) = Theta(log log n)`。这类题不要硬套主定理。

分治题还常考比较下界。几个结论要直接记住：

- 找最大值需要 `n - 1` 次比较。
- 同时找最大值和最小值需要约 `3n/2 - 2` 次比较。
- 找次大值可以用锦标赛树，只需比较输给最大值的那一串候选。
- 基于比较的排序下界是 `Omega(n log n)`。

## 动态规划

动态规划题最常见的失分点不是不会递推，而是状态没定义清楚。写题时先问三个问题：

- 最优子结构是什么？
- 子问题的规模按什么顺序递增？
- 如何从表中恢复解？

区间 DP、树形 DP、背包、编辑距离、矩阵链乘、石子合并都属于典型考法。期末答题里建议把递推写成“枚举最后一步”或“枚举断点”的形式，这样正确性更容易解释。

背包问题要特别注意输入规模。0-1 背包有动态规划算法，但它依赖容量数值，是伪多项式时间算法；这不意味着 0-1 背包不属于 NP 完全问题。

## 网络流与线性规划

网络流题的核心是建模。复习时不要只背 Ford-Fulkerson 或 Edmonds-Karp 的复杂度，更要熟悉哪些约束可以翻译成容量：

- “每个对象最多选一次”通常用单位容量边。
- 二部图匹配可以建成源点到左部、左部到右部、右部到汇点的单位容量网络。
- “每类至少多少、至多多少”会引出带上下界流。
- “选择对象之间存在兼容关系”常常可以通过中间层节点表达。

最大流最小割定理是证明题和选择题的基础。一个流的值等于某个割的容量时，二者同时最优。

线性规划题常见要求是把自然语言约束写成变量、目标函数和线性不等式。整数规划通常是 NP 难的，LP 松弛加舍入则是设计近似算法的重要套路。

## 摊还分析

摊还分析研究的是一串操作的总成本，而不是单次最坏成本。三种常见方法要会互相翻译：

- 聚合法：直接证明 `n` 次操作总成本是 `O(f(n))`。
- 记账法：便宜操作预先多付一点，用来抵消未来昂贵操作。
- 势能法：定义数据结构的势能，真实成本加势能变化得到摊还成本。

动态数组扩容、二进制计数器翻转、栈的多重弹出都是典型例子。关键是说明势能函数非负，并且初始势能通常为 0，这样摊还总成本才能上界真实总成本。

## 随机算法

随机算法先分清随机性影响什么。

Las Vegas 算法一定返回正确结果，随机性影响运行时间。随机快排和随机选择属于这一类。Monte Carlo 算法通常有固定时间上界，但可能出错；指纹法、随机主元素测试等题目常出现在这里。

复习时要把三个判断标准写在一起：

1. 结果是否一定正确？
2. 时间是否有确定上界？
3. 错误能否被验证并重试？

如果一个 Las Vegas 算法有期望时间 `O(n^2)`，可以通过截断运行并在超时时任意输出，把它转成一个 `O(n^2)` 时间的 Monte Carlo 算法。这类转化常用 Markov 不等式证明。

## 近似算法

近似算法题通常有固定三步：

1. 描述算法。
2. 证明算法解和 `OPT` 的比例关系。
3. 构造紧实例说明这个比例不能轻易改进。

几个模板要熟：

- 顶点覆盖：每次取一条未覆盖边，把两个端点都加入覆盖，得到 2-近似。
- 多机调度 List Scheduling：把作业按顺序分配给当前最早空闲机器，近似比是 `2 - 1/m`。
- 度量 TSP：基于 MST 的双生成树算法给出 2-近似；Christofides 给出 `3/2`-近似。
- 一般 TSP 没有常数近似算法，除非 `P = NP`。
- MAX-3-CNF 随机赋值期望满足 `7/8` 的子句。

近似比不是越大越好。对最小化问题，近似比越接近 1 越好。

## NP 完全性证明

NP 完全性是最应该模板化的部分。证明一个问题 `X` 是 NP 完全，一般写：

1. `X in NP`：给定一个候选解，可以在多项式时间内验证。
2. `X` 是 NP-hard：选择一个已知 NP 完全问题 `Y`，构造多项式时间变换，使 `Y` 有解当且仅当 `X` 有解。

归约方向必须是

```text
Known-NPC <=p X
```

而不是把新问题归约到已知问题。常用起点包括 SAT、3-SAT、顶点覆盖、独立集、团、哈密顿路径、TSP、子集和、划分、多机调度。

几个客观题结论也要分清：

- `P = NP` 会推出每个 NP 完全问题都可多项式求解。
- NP-hard 不一定在 NP 中，所以不一定是 NP-complete。
- 一个问题要是 NP-complete，必须既在 NP 中，又是 NP-hard。
- P 类问题也属于 NP。

## 考前冲刺顺序

如果时间有限，可以按下面顺序复习：

1. NP 完全性证明：背熟归约方向和常用等价关系。
2. 递归方程：主定理、变量替换、递归树。
3. 近似算法：顶点覆盖、多机调度、度量 TSP。
4. 网络流建模：二部图匹配、上下界、最小割。
5. 动态规划：状态、递推、边界、标记函数。
6. 随机算法：Las Vegas / Monte Carlo 区分和期望分析。
7. 摊还分析：聚合、记账、势能三种证明。
