## 分治策略的核心思想

分治法可以概括成一句话：

把一个难以直接求解的大问题，拆成若干个规模更小、形式相同的子问题，递归求解后再合并答案。

标准流程通常分成三步：

1. Divide：把原问题拆成若干子问题
2. Conquer：递归求解子问题，规模足够小时直接求
3. Combine：把子问题答案合并成原问题答案

写题时最好把这三步明确写出来，而不是只写“用分治”。

## 分治算法通用模板

```text
Divide-and-Conquer(P):
1. 若 |P| 足够小，直接求解
2. 将 P 分解成若干个更小的子问题 P1, P2, ..., Pk
3. 递归求解每个子问题
4. 合并各子问题的解
5. 返回结果
```

判断一个问题适不适合分治，重点看三件事：

- 子问题是否和原问题同构
- 子问题之间是否基本独立
- 合并步骤是否足够便宜

如果子问题有大量重叠，通常更像动态规划而不是分治。

## 分治复杂度分析

分治法最常见的分析工具是递推方程。

### 1. 均匀划分型

```text
T(n) = aT(n/b) + f(n)
```

其中：

- `a` 是子问题个数
- `n/b` 是每个子问题规模
- `f(n)` 是划分与合并的代价

这一类可以优先考虑主定理。

### 2. 非均匀划分型

```text
T(n) = T(n-1) + f(n)
T(n) = T(alpha n) + T((1-alpha)n) + f(n)
```

这一类更适合用迭代法或递归树。

## 主定理的复习框架

对于

```text
T(n) = aT(n/b) + f(n)
```

先比较 `f(n)` 与 `n^(log_b a)` 的增长速度。

常见三种情形：

1. `f(n)` 更小，则主体代价来自递归树叶子，结论通常是 `Theta(n^(log_b a))`
2. `f(n)` 同阶，则多出一个 `log n`，结论通常是 `Theta(n^(log_b a) log n)`
3. `f(n)` 更大且满足正则条件，则结论通常由 `f(n)` 主导

考试里最重要的不是背模板，而是先判断属于哪一类，再说明理由。

## 分治法的关键设计点

### 子问题独立性

分治默认子问题彼此独立。如果求一个子问题时要反复依赖另一个子问题的中间结果，分治就不一定自然。

### 划分是否平衡

划分越平衡，递归树越规整，复杂度通常越好。

- 归并排序每次均分，复杂度稳定为 `O(n log n)`
- 快速排序如果每次划分极不平衡，最坏会退化到 `O(n^2)`

### 合并代价

分治算法能否高效，往往取决于合并步骤。

- 归并排序的关键在于合并只需 `O(n)`
- 最近点对问题的关键在于经过预处理后，跨中线检查可以压到线性量级

## 典型例子

### 1. 二分查找

问题规模每次减半，递推式为：

```text
T(n) = T(n/2) + O(1)
```

因此复杂度为 `O(log n)`。

这是最经典的“只分不并”型分治。

### 2. 归并排序

把数组分成两半，各自排好序，再线性合并：

```text
T(n) = 2T(n/2) + O(n)
```

由主定理得：

```text
T(n) = O(n log n)
```

特点：

- 稳定排序
- 额外空间开销较大
- 复杂度稳定，不会退化

### 3. 快速排序

核心思路是选一个 pivot，把数组划成左小右大两部分，再递归处理。

均衡时：

```text
T(n) = 2T(n/2) + O(n) = O(n log n)
```

最坏时：

```text
T(n) = T(n-1) + O(n) = O(n^2)
```

所以快速排序不是“分治一定优”，而是“分治 + 划分质量决定效率”。

### 4. 快速幂

利用

```text
a^n =
  (a^(n/2))^2, n 为偶数
  (a^((n-1)/2))^2 * a, n 为奇数
```

递推式：

```text
T(n) = T(n/2) + O(1)
```

复杂度是 `O(log n)`，比连乘 `n` 次的 `O(n)` 快很多。

### 5. Strassen 矩阵乘法

普通分块矩阵乘法的递推式是：

```text
T(n) = 8T(n/2) + O(n^2)
```

复杂度为 `O(n^3)`。

Strassen 通过代数变换，把 8 次子矩阵乘法降成 7 次：

```text
T(n) = 7T(n/2) + O(n^2)
```

因此复杂度降到：

```text
O(n^(log_2 7))
```

这一例子说明：分治优化不只来自“拆”，还来自“减少子问题数”。

### 6. 最近点对

暴力法需要检查所有点对，复杂度 `O(n^2)`。

分治做法一般是：

1. 先按 `x` 坐标排序
2. 分成左右两部分
3. 递归求左右部分最近点对
4. 再处理跨中线的情况

关键不在于“分”，而在于预处理后跨中线部分仍能在线性时间里完成，所以总复杂度可做到 `O(n log n)`。

## 容易混淆的点

### 分治 vs 动态规划

两者都在拆子问题，但侧重点不同：

- 分治：子问题独立，递归后再合并
- 动规：子问题重叠，重用中间结果

如果题目里同一个子问题会被反复算到，就要警惕它其实更像动态规划。

### 分治 vs 回溯

- 分治强调把原问题拆成并列子问题
- 回溯强调在解空间树上试探、剪枝、撤销

前者通常在“所有子问题都要做”；后者通常在“尝试路径，发现不行就退回”。

## 复习抓手

考场上如果看见一道分治题，优先按下面顺序写：

1. 子问题如何划分
2. 递归关系是什么
3. 合并过程做什么
4. 递推式怎么列
5. 复杂度如何求

常见高频结论：

| 算法 | 递推式 | 复杂度 |
| --- | --- | --- |
| 二分查找 | `T(n)=T(n/2)+O(1)` | `O(log n)` |
| 归并排序 | `T(n)=2T(n/2)+O(n)` | `O(n log n)` |
| 快速排序（均衡） | `T(n)=2T(n/2)+O(n)` | `O(n log n)` |
| 快速排序（最坏） | `T(n)=T(n-1)+O(n)` | `O(n^2)` |
| 快速幂 | `T(n)=T(n/2)+O(1)` | `O(log n)` |
| Strassen | `T(n)=7T(n/2)+O(n^2)` | `O(n^(log_2 7))` |

## 一句话总结

分治法最核心的两件事是：

- 会把问题拆成“同构、独立、规模更小”的子问题
- 会把复杂度写成递推式，并用递推工具把它真正算出来
