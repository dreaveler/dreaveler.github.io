> 这篇笔记把角色动画课程后段的两条线接起来：一条是“怎样用肌肉而不是关节电机驱动角色”，另一条是“角色如何从当前位置规划到目标位置并避开障碍”。前者解释力从哪里来，后者解释角色要往哪里走。

## 从关节力矩到肌肉激活

前面的物理角色控制常把控制器输出写成关节力矩 `tau`。这很像给每个关节装上电机：控制器直接决定关节该受多大力矩，仿真器负责把力矩积分成下一帧姿态。

肌肉驱动模型换了一个更接近生物的接口。控制器不直接给 `tau`，而是给每块肌肉一个激活水平：

```text
a in [0, 1]
```

激活信号经过肌肉-肌腱动力学，先变成肌肉张力，再通过肌肉路径和骨骼几何映射成关节力矩。完整链路是：

```text
神经命令 -> 肌肉激活 -> 肌肉/肌腱张力 -> 关节力矩 -> 多体动力学 -> 下一帧姿态
```

这个接口更难控制，但能表达很多关节力矩模型不自然的现象，例如拮抗肌共同激活带来的关节刚度、肌肉长度和收缩速度对产力能力的限制，以及肌腱弹性带来的缓冲。

## 肌肉-肌腱单元

仿真中通常把一块肌肉和两端肌腱合并成 muscle-tendon unit，简称 MTU。MTU 被抽象成一条一维路径：它从一块骨骼的附着点出发，可能绕过关节或包裹物，最后连接到另一块骨骼。

肌肉只能拉，不能推。为了实现双向运动，人体依靠拮抗肌对。例如屈肘时肱二头肌是主动肌，肱三头肌是拮抗肌；伸肘时角色互换。两者同时激活时，关节更稳定，但能量消耗更高。

肌纤维并不总是与肌腱方向平行。羽状角 `alpha` 描述肌纤维和肌腱方向的夹角。肌纤维产生的力要乘上 `cos(alpha)` 才是沿肌腱方向的有效张力。

## Hill 型肌肉模型

Hill 型模型把肌肉抽象成三个力学元件：

| 元件 | 含义 |
| --- | --- |
| CE | contractile element，主动收缩元，受激活、长度、速度调制 |
| PE | parallel elastic element，并联弹性元，表示肌肉被拉长后的被动弹性 |
| Tendon | 串联肌腱，表示传力和储能的弹性肌腱 |

肌肉力可以写成：

```text
f^M = fbar_o * [a * f_L(l^M) * f_V(v^M) + f_PE(l^M)]
```

其中 `fbar_o` 是最大等长张力，`a` 是激活水平，`f_L` 是力-长度曲线，`f_V` 是力-速度曲线，`f_PE` 是被动弹性项。

这条公式的直觉是：

- 肌肉在最佳长度附近最有力，太短或太长都会弱。
- 肌肉快速缩短时产力下降，被拉长时可以产生更大阻力。
- 即使没有主动激活，肌肉被拉过自然长度后也会有被动张力。

肌腱也有自己的力-长度曲线。低于松弛长度时几乎不产力，被拉长后像非线性弹簧一样产生张力。

## 收缩动力学与激活动力学

给定当前姿态和激活值，系统需要解出肌肉纤维长度、肌腱长度、纤维速度和最终张力。关键约束包括：

```text
f^T = f^M * cos(alpha)
l^MT = l^T + l^M * cos(alpha)
```

第一条是肌肉力和肌腱力的平衡，第二条是 MTU 总长度由肌腱长度和肌肉纤维在肌腱方向的投影组成。MTU 总长度由当前骨骼姿态和肌肉路径几何决定。

真实肌肉激活也不是瞬时变化。神经命令 `u` 会经过一阶滤波变成实际激活 `a`。激活通常比去激活更快，例如激活时间常数可以取 10ms，去激活时间常数可以取 40ms。这会让动作更平滑，也让控制策略不能靠瞬间开关肌肉作弊。

## 从肌肉张力到关节力矩

肌肉张力是标量，但作用到骨骼上必须变成力和力矩。做法是沿 MTU 路径两端的切向方向施加拉力，再根据力臂计算相对各关节的广义力矩。

如果一条肌肉跨过膝关节，它的张力会对膝关节产生力矩；如果它同时跨过髋关节，也会影响髋关节。这就是多关节肌肉的耦合效应。相比直接控制每个关节的 `tau`，肌肉控制的动作空间更受生理结构约束。

OpenSim 等工具可以从解剖模型中导入肌肉路径、骨骼结构和参数。学习式方法如 Muscle VAE 则会把高维肌肉激活压缩成更容易控制的低维潜变量，用于生成自然动作。

## 运动规划问题：从 Workspace 到 C-space

肌肉模型回答“角色怎样产生力”，运动规划回答“角色应该往哪里走”。规划问题通常给定起点、目标和障碍，要求找一条不碰撞路径。

在工作空间中，障碍是几何物体，角色也是有形状的物体。直接处理“复杂角色是否碰到复杂障碍”很麻烦。构型空间把角色压成一个点：

```text
q = 角色的完整位姿参数
```

例如平面机器人可以用 `(x, y, theta)` 表示，机械臂可以用所有关节角表示，人形角色则包含根位置、根朝向和各关节角。

在 C-space 中：

```text
C_obstacle = 会发生碰撞的所有构型
C_free     = C \ C_obstacle
```

规划就变成：在 `C_free` 里找一条从 `q_init` 到 `q_goal` 的连续路径。

## 图搜索：BFS、UCS 与 A*

如果 C-space 已经离散成图，规划就退化成图搜索。常用算法差别主要在优先级。

| 算法 | 优先级 |
| --- | --- |
| BFS | 扩展步数少的节点 |
| DFS | 沿一条路尽量深入 |
| UCS | 扩展起点到当前代价 `g` 最小的节点 |
| A* | 扩展 `f = g + h` 最小的节点 |

A* 的关键是启发函数 `h`。若 `h(x)` 从不高估 `x` 到目标的真实最短代价，它就是 admissible 的；此时 A* 能保证找到最优路径。欧氏距离通常是最短路问题的安全启发式，因为直线距离不会超过绕行距离。

## 几何路标：Bug、网格、可见图与单元分解

几何规划方法会先构造一张代表自由空间连通性的图，再在图上搜索。

Bug 算法只依赖局部碰撞感知。角色朝目标直走，撞到障碍后沿边界走，找到合适离开点再继续。它路径冗余，但展示了完备性的基本思想。

Grid Search 把空间切成网格。网格越细越接近连续空间，但节点数随维度指数增长。高自由度角色上，均匀网格很快不可用。

Visibility Graph 适用于二维多边形障碍。节点是起点、终点和障碍顶点；若两点之间线段不穿过障碍，就连边。二维最短路径的拐点可以落在障碍顶点上，所以可见图能捕捉最短路。三维中这个性质通常失效。

Cell Decomposition 把自由空间切成简单单元，使单元内连线容易保证无碰撞，再连接相邻单元形成图。

## 采样规划：PRM、RRT 与最优版本

高维 C-space 中，显式建完整地图很难。采样方法改为随机抽取构型，只保留无碰撞点和可连接边。

PRM 的流程是：

1. 在 `C_free` 中采样许多构型。
2. 把相近样本用局部规划器连接。
3. 得到一张 roadmap。
4. 查询时把起点和目标接入图，再做图搜索。

PRM 适合多次查询同一环境，因为 roadmap 可以复用。

RRT 则从起点长出一棵树。每次随机采样一个点，找到树上最近节点，朝采样点方向扩展一小步。如果新节点无碰撞，就加入树。RRT 天然偏向尚未覆盖的空间，适合快速找到一条可行路径。

PRM 和 RRT 通常是概率完备的：如果解存在，采样次数趋于无穷时找到解的概率趋于 1。PRM* 和 RRT* 进一步加入更严格的连接或 rewiring，使路径代价渐近收敛到最优。

## 群体仿真：Boids 与社会力

单个角色规划扩展到多人时，目标不只是避开静态障碍，还要避免角色之间互相碰撞，并表现出群体行为。

Boids 模型用三条局部规则生成群体运动：

| 规则 | 含义 |
| --- | --- |
| Separation | 离太近就分开 |
| Alignment | 朝向和邻居保持一致 |
| Cohesion | 向邻居中心靠拢 |

Social Force Model 则把行人看作受力粒子。每个人有朝目标前进的驱动力，也有来自其他人和障碍的排斥力。它能描述拥挤、绕行、通道瓶颈等群体现象。

## 两条线怎样接起来

运动规划给出的是期望路径或目标速度，肌肉/动力学控制负责把这个意图变成身体动作。对游戏角色，可以用规划器决定走廊中的路径，再用 motion matching 或控制器生成动作。对物理角色，可以把规划输出变成目标速度、落脚点或奖励函数，再由关节力矩或肌肉激活策略执行。

因此课程后段的结构可以这样理解：

```text
高层规划：我要到哪里，怎样绕开障碍
中层控制：下一步速度、落脚点、姿态目标是什么
底层动力学：用关节力矩或肌肉激活产生身体运动
```

肌肉驱动让“身体怎样动”更真实，运动规划让“角色往哪里动”更合理。两者合起来，才是面向交互场景的角色运动系统。
